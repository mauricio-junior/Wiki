이 릴리스 노트는 위키 문서이므로 커뮤니티 구성원들에 의해 추가 혹은 변경될 수 있습니다. 릴리스 노트는 면밀하고 신중하게 점검됩니다. 변경하신 사항은 곧바로 승인되거나 다듬어지며, 경우에 따라 취소될 수도 있습니다. 원본 공식 릴리스 노트를 보려면 여기를 클릭하십시오.

이 릴리스 정보 문서에서는 SQL Server 2008 R2 SP1을 설치하거나 문제를 해결하기 전에 읽어야 할 알려진 문제에 대해 설명합니다. SQL Server 2008 R2 서비스 팩 1(다운로드하려면 여기를 클릭)을 다운로드할 수 있습니다. 또한 SQL Server 2008 R2에 대한 Microsoft 지원 사이트의 KB(정보 자료) 문서에서 알려진 문제를 추가로 찾기 위한 검색 팁이 나와 있습니다. 이 릴리스 정보 문서와 KB 문서에는 SQL Server 2008 R2 온라인 설명서를 보완하는 추가 정보가 들어 있습니다.

이 릴리스 정보 문서는 정기적으로 업데이트되며 온라인으로만 사용할 수 있고 설치 미디어에는 포함되지 않습니다.

SQL Server 2008 R2 설치를 시작하는 방법은 SQL Server 2008 R2 추가 정보를 참조하십시오. 추가 정보는 설치 미디어에 포함되어 있거나 추가 정보 다운로드 페이지에서 다운로드할 수 있습니다.

SQL Server 2008 R2에 대한 사용자 의견이 있는 경우 Microsoft SQL Server Connect의 사용자 의견 센터로 보내 주십시오. 최신 설명서를 보려면 MSDN Library에서 SQL Server 2008 R2 온라인 설명서를 참조하십시오. 문제 해결 및 지원 페이지에서 포럼 및 기타 지원도 찾을 수 있습니다.

1.0 서비스 팩 1의 새로운 기능

다음은 SP1(서비스 팩 1)의 새로운 기능입니다.

  • 지원 가능성 향상을 위한 동적 관리 뷰. 장기 실행 쿼리 문제 해결에 대한 지원 가능성을 향상시키기 위해 sys.dm_query_stats DMV가 추가 열과 함께 확장되었습니다. 성능 카운터 선택을 위한 새 DMV 및 XEvents가 도입되어 SQL Server 인스턴스와 관련한 OS 구성 및 리소스 상태를 모니터링할 수 있습니다.

  • 쿼리 성능 향상을 위한 ForceSeek. 선택적 매개 변수를 사용하여 인덱스에 대한 액세스 방법을 더 구체적으로 제어할 수 있도록 FORCESEEK 인덱스 힌트에 대한 구문이 수정되었습니다. FORCESEEK에 대한 이전 스타일의 구문 사용은 수정되지 않았으며 이전과 마찬가지로 작동합니다. 이와 함께 새 쿼리 힌트인 FORCESCAN이 추가되었습니다. 이는 인덱스에 액세스하는 방법으로 'scan'을 지정할 수 있도록 하여 FORCESEEK 힌트를 보완합니다. 이러한 새 기능이 필요하지 않다면 응용 프로그램을 변경할 필요가 없습니다.

  • 데이터베이스 업그레이드 향상을 위한 데이터 계층 응용 프로그램 구성 요소 프레임워크(DAC Fx). 새 DAC(데이터 계층 응용 프로그램) 프레임워크 v.1.1 및 DAC 업그레이드 마법사로 데이터베이스 스키마 관리를 위한 새 전체 업그레이드 서비스가 가능해집니다. 새 전체 업그레이드 서비스는 SQL Azure 및 DAC가 지원하는 SQL Server 버전의 기존 데이터베이스에 대한 스키마를 업그레이드합니다. DAC는 응용 프로그램에 사용되는 모든 데이터베이스 개체 및 인스턴스 개체가 포함된 엔터티이며, 데이터 계층 개체를 제작, 배포 및 관리하기 위한 단일 단위를 제공합니다. 자세한 내용은 데이터 계층 응용 프로그램 디자인 및 구현을 참조하십시오.

  • PowerPivot for SharePoint의 디스크 공간 제어. 이 업데이트는 캐시된 데이터를 얼마나 오래 시스템에 저장할 것인지를 결정할 수 있는 두 가지 새로운 구성 설정을 제공합니다. PowerPivot 구성 페이지의 새 섹션인 디스크 캐시에서는 비활성화된 데이터베이스를 언로드하기 전eSeek. 선택적 매개 변수를 사용하여 인덱스에 대한 액세스 방법을 더 구체적으로 제어할 수 있도록 FORCESEEK 인덱스 힌트에 대한 구문이 수정되었습니다. FORCESEEK에 대한 이전 스타일의에 메모리에 얼마나 오래 유지할지를 지정할 수 있습니다. 또한 캐시된 파일이 삭제되기 전에 디스크에 얼마나 오래 저장해둘지도 제한할 수 있습니다.

  • 여러 가지 문제점 수정. 이 업데이트에서는 이 마스터 KB 문서에 등록되어 있는 SQL Server 2008 R2 문제 ��수가 수정되었습니다. 제품 품질을 향상시키는 일부 추가 수정도 포함되어 있습니다.

2.2 UCP가 SQL Server 인스턴스에 있을 경우 SQL Server 2008 R2 SP1 제거 이후 SQL Server 서비스를 시작할 수 없음

증상

다음과 같은 시나리오를 생각해 볼 수 있습니다.

  • UCP(유틸리티 제어 지점)를 호스팅하는 Microsoft SQL Server 2008 R2 인스턴스가 있습니다.
  • SQL Server 2008 R2 SP1을 인스턴스��� 설치했다가 나중에 제거합니다.

이 시나리오에서 제거는 성공합니다. 하지만 SQL Server 서비스는 시작할 수 없습니다.

원인

이 문제는 SQL Server 2008 R2가 msdb 데이터베이스에 대해 업그레이드 스크립트를 실행할 때 에이전트 XPs 옵션이 설정되지 않았기 때문에 발생합니다.

업그레이드 스크립트는 SQL Server 서비스가 시작될 때 실행됩니다. 이 스크립트는 UCP와 관련되어 있는 저장 프로시저를 실행합니다. 이 저장 프로시저를 실행하려면 에이전트 XPs 옵션을 사용하도록 설정해야 합니다.

해결 방법

SP1을 제거한 이후에 이 동작이 발생한 경우 다음 단계를 수행하여 문제를 해결합니다. 문제를 방지하려면 SP1을 제거하기 전에 다음 단계를 수행합니다.

1) SQL Server가 실행되고 있으면 SQL Server 서비스를 중지합니다.

2) SQL Server 2008 R2 인스턴스에서 추적 플래그 902를 설정합니다. 이렇게 하려면 다음 단계를 수행합니다.

  • SQL Server 구성 관리자를 엽니다.
  • SQL Server 구성 관리자에서 SQL Server 서비스를 클릭합니다.
  • SQL Server 서비스를 두 번 클릭합니다.
  • SQL Server 속성 대화 상자에서 고급 탭을 클릭합니다.
  • 고급 탭에서 시작 매개 변수 항목을 찾습니다.
  • 기존 문자열 값의 끝에 ;–T902를 추가한 다음 확인을 클릭합니다.

3) SQL Server 서비스를 마우스 오른쪽 단추로 클릭한 다음 시작을 클릭합니다.

4) SQL Server 에이전트 서비스가 실행되고 있으면 SQL Server 에이전트 서비스를 마우스 오른쪽 단추로 클릭한 다음 중지를 클릭합니다. 에이전트 서비스 시작 모드를 수동으로 설정합니다.

5) SQL Server Management Studio를 연 다음 SQL Server 2008 R2 인스턴스에 연결합니다.

6) 다음 문을 실행합니다.

EXEC sp_configure 'show advanced', 1;
RECONFIGURE;
EXEC sp_configure 'allow updates', 0;
RECONFIGURE;
EXEC sp_configure 'Agent XPs', 1;
RECONFIGURE;
GO

7) sysutility_mdw 데이터베이스의 확장된 속성을 SQL Server 2008 R2 버전으로 다시 수정합니다. 이렇게 하려면 다음 단계를 수행합니다.

  • SQL Server Management Studio에서 sysutility_mdw 데이터베이스를 마우스 오른쪽 단추로 클릭한 다음 속성을 클릭합니다.
  • 속성 대화 상자에서 확장 속성을 클릭합니다.
  • “Microsoft_DataCollector_MDW_Version” 속성 값을 10.50.1600.1로 업데이트합니다.

8) 서비스 팩을 아직 제거하지 않은 경우 지금 제거합니다. 이미 제거한 경우 9단계를 진행합니다.

9) SQL Server 구성 관리자에서 SQL Server 서비스를 마우스 오른쪽 단추로 클릭한 다음 중지를 클릭합니다.

10) SQL Server 2008 R2 인스턴스에서 추적 플��그 902를 제거합니다. 이렇게 하려면 2단계에서 업데이트한 문자열 값에서 ;–T902를 삭제합니다.

11) SQL Server 서비스를 마우스 오른쪽 단추로 클릭한 다음 시작을 클릭합니다.

12) SQL Server 에이전트 서비스를 마우스 오른쪽 단추로 클릭하고 속성을 표시하고 서비스 시작 모드를 원래 값으로 다시 설정합니다. 그런 다음 시작을 클릭하여 에이전트 서비스를 시작합니다.

13) SQL Server Management Studio에서 SQL Server 2008 R2 인스턴스에 다시 연결합니다.

14) 개체 탐색기에서 관리를 확장하고 데이터 컬렉션을 마우스 오른쪽 단추로 클릭한 다음 데이터 컬렉션 사용을 클릭합니다.

참고: 데이터 컬렉션이 이미 활성화된 경우 데이터 컬렉션 사용 항목은 사용할 수 없습니다.

3.0 알려진 문제를 추가로 찾는 방법

SQL Server 2008 R2에 대한 Microsoft 지원 사이트 또는 기본 Microsoft 고객 지원 웹 사이트에서 "SQL Server 2008 R2"를 검색하여 알려진 문제를 더 많이 볼 수 있습니다.

SQL Server 커뮤니티를 통해 다른 SQL Server 2008 사용자들과 경험을 공유하고 궁금한 내용을 물어볼 수 있습니다. 자세한 내용은 SQL Server 커뮤니티 웹 사이트를 참조하십시오. 문제 해결 및 지원 페이지에서 포럼 및 기타 지원도 찾을 수 있습니다.
