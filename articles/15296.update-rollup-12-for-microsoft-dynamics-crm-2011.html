Release schedule

Update Rollup 12 for on-premises CRM servers was scheduled to be released on January 10th, 2013, but it was removed shortly after publishing during the same day due to an issue discovered and later re-released as a new v2 edition on January 29th, 2013. CRM Outlook clients update was released earlier on January 8th. UR12 was scheduled to become available through Microsoft Update on January 22nd, 2013, but only the Outlook, email router and reporting extensions files were released, while the server bits remain unavailable. The build number of the original on-premise release was 5.0.9690.3218. The v2 files published through Microsoft Update have a newer build number 5.0.9690.3233. CRM Online update schedule is tied to the roll-out of December 2012 Service Update. Existing CRM Online organizations began receiving the update in the beginning of January 2013. New trial sign-ups will have the update starting from mid-January. It is important to note that Update Rollup 12 server bits do not deliver the same features as December 2012 Service Update (codename Polaris) for CRM Online. The new features, such as the new "flow UI", Bing Maps & Skype integration, iPad Safari client, Activity Feeds update and Yammer integration, will not be available for on-premises environments until the next release (codename Orion) expected to arrive in Q2 2013.

Highlights

  • Update Rollup 12 introduces the cross-browser support that was originally scheduled for the Q2 2012 Service Update but later postponed. See the following KB article for details: Browser compatibility introduced with Microsoft Dynamics CRM 2011 Update Rollup 12 or the December 2012 Service Update.
  • ExecuteMultiple request for improving bulk data load performance. See the following SDK article for details: Use ExecuteMultiple to Improve Performance for Bulk Data Load.
  • New indexes for selected entities for the Quick Find Search Optimization feature. See article KB2792837.

Potential issues

  • With introduction of Multi browser support in UR12, existing Form Scripts could break. More details on the Microsoft Blog Article: Script Errors After the December 2012 Service Update
  • Dynamics CRM Online with December 2012 Service Update is not compatible with the non-multi browser version of SharePoint List Components. More details on the Microsoft Blog Article: Errors while using Documents on SharePoint
  • Reports displayed in the report viewer now render differently than they did prior to this update (increased spacing around elements etc...)
  • Printing from the report viewer is only available in Internet Explorer
  • When using a dashboard chart with a View Selector enabled, the chart disappears after changing the view (screenshot)
  • CRM Online: Notes on forms (process & classic) no longer show created on & created by information, printing the form only displays first row of note contents (forum post)
