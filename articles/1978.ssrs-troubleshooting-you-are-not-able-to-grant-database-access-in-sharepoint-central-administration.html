Background

On the Reporting Services integration page in SharePoint Central Administration you see an error message indicating the grant database access step failed.

This issue occurs when SharePoint Central Administration and the report server are running on the same computer and the user credentials used for logging into WMI are the same as the application pool account that is used by SharePoint Central Administration.

Workaround

When you grant the database access from SharePoint Central Administration, use a login from the local administrators group and make sure that this login is not also the Application pool account for the Central Application Web application.

Applies to

  • Windows SharePoint Services 3.0
  • Office SharePoint Server 2007
