Apa itu Dynamic Memory?

Dynamic Memory merupakan fitur baru dari Microsoft Hyper-V yang membantu pengguna menggunakan fisikal memory secara lebih efisien. Dengan Dynamic Memory, Hyper-V memperlakukan memory sebagai resource yang dibagi yang dapat diatur ketika virtual mesin sedang aktif. Dynamic Memory mengatur jumlah memori yang tersedia untuk virtual mesin berdasarkan perubahan kebutuhan yang anda tentukan. Untuk dapat memanfaatkan fitur Dynamic Memory, anda harus menginstal Windows Server 2008 R2 Service Pack 1, karena fitur tersebut membutuhkan Windows Server 2008 R2 Service Pack 1.

Gambar Arstektur Dynamic Memory pada Windows Server 2008 R2 SP1

Untuk informasi lebih lanjut tentang cara menginstall service pack tersebut, baca lebih lanjut pada panduan penginstalan Windows Server R2 Service Pack 1 dan Windows 7 dengan SP1 (http://go.microsoft.com/fwlink/?linkId=192441).


Apa Keuntungan dari Dynamic Memory?

Dynamic Memory membantu anda menggunakan resource lebih efisien. Sebelum fitur ini ada, untuk mengubah jumlah memory pada virtual mesin, anda harus mematikan virtual mesin terlebih dahulu. Dengan Dynamic Memory, Hyper-V dapat menyediakan virtual mesin dengan memory dinamis dalam merespon perubahan jumlah kebutuhan memory dengan beban atau aplikasi yang berjalan diatas virtual mesin. Sebagai hasilnya, Hyper-V dapat mendistribusikan memory lebih efisien terhadap virtual mesin yang diatur dengan Dynamic Memory. Tergantung pada factor seperti beban kerja, efisiensi ini memungkinkan untuk menjalankan virtual mesin lebih banyak pada waktu yang sama pada satu komputer (host).


Apa prasyarat untuk menggunakan fitur Dynamic Memory?

 Untuk menggunakan fitur Dynamic Memory, anda membutuhkan hal berikut:
  • Server yang manjalankan Hyper-V, dengan Windows Server 2008 R2 Service Pack 1. Untuk petunjuk tentang cara menginstall SP1, lihat Petunjuk penginstalan untuk Windows Server 2008 R2 dengan SP1 dan Windows 7 dengan SP1 (http://go.microsoft.com/fwlink/?LinkID=192441)
  • Virtual mesin yang menjalankan system operasi guest yang mendukung Dynamic Memory. Untuk mengaktifkan Dynamic Memory, virtual mesin harus menjalankan satu dari sistem operasi guest dan mempunyai konfigurasi seperti pada tabel berikut.

Catatan

  • Sistem Ooperasi Host (host) adalah sistem operasi yang diinstal pada komputer fisik dimana virtual mesin dijalankan (diinstal)
  • Sistem Operasi Guest (guest) adalah sistem operasi yang diinstal pada virtual mesin

Dengan adanya fitur Dynamic Memory pada Hyper-V, akan sangat membantu para pengguna virtual mesin yang menjalankan virtual mesin diatas Windows Server 2008 R2 Hyper-V dalam mengatur penggunaan memori secara lebih efisien tanpa harus mematikan sistem operasi guest sehingga dapat mengurangi downtime pada lingkungan virtual mesin mereka.
