2.0 알려진1.0 설치하기 전

PowerPivot을 설치하기 전에 다음과 같은 필요한 동작을 검토하십시오.

1.1 VSTO 4.0 런타임이 설치되어 있지 않으면 PowerPivot을 설치할 수 없습니다.

PowerPivot을 설치하기 전에 Visual Studio Tools for Office(VSTO) 4.0 런타임을 반드시 설치해야 합니다. Microsoft 다운로드 센터 페이지 에서 VSTO 런타임을 다운로드할 수 있습니다. 이 문제는 Office 2010 사용자에만 해당합니다. Office 2010 SP1가 있습니다. 이 문제는 Office 2010 사용자에만 해당합니다. Office 2010 SP1가 릴리스되면 개별 VSTO 런타임 다운로드가 필요하지 않습니다.

1.2 컴퓨터에 .NET Framework 4.0이 필요합니다.

사용자의 컴퓨터에 .NET Framework 4.0이 설치되어 있지 않은 경우 PowerPivot for Excel의 SQL Server 2012 버전이 성공적으로 설치되더라도 Excel 리본에 PowerPivot 탭이 나타나지 않습니다. .NET Framework 4.0을 반드시 설치해야 합니다.

1.3 컴퓨터에 Silverlight 5가 필요합니다.

SharePoint용 PowerPivot 및 Power View에는 사용자의 컴퓨터에 Silverlight 5를 설치해야 하는 Silverlight 기반 구성 요소가 포함되어 있습니다. 필요한 Silverlight 버전이 설치되어 있지 않으면 Silverlight이 필요한 웹 응용 프로그램 영역으로 이동할 때 Silverlight를 설치하라는 메시지가 표시됩니다. http://go.microsoft.com/fwlink/?LinkId=243096에서 Silverlight 5를 설치할 수 있습니다.

1.4 SharePoint용 PowerPivot을 사용하려면 SharePoint Server 2010 SP1이 필요합니다.

SharePoint 환경에서 SQL Server 2012 기능을 사용하려는 경우 SharePoint Server 2010 SP1(서비스 팩 1)은 필수입니다. SharePoint Server 2010 SP1의 최신 패치를 설치하는 것이 좋습니다.

해결 방법: SQL Server 기능을 팜에 추가하기 전에 SP1을 설치하고 서버 팜을 완전히 패치해야 합니다. 이러한 요구 사항은 다음의 SQL Server 2012 기능에 적용됩니다.

· 팜의 데이터베이스 서버로 데이터베이스 엔진의 인스턴스 사용

· SharePoint용 PowerPivot 구성

· SharePoint 모드에서 Reporting Services 배포

2.0 알려진 문제

Microsoft SharePoint용 SQL Server 2012 PowerPivot에는 다음과 같은 문제점들이 알려져 있습니다.

2.1 '가능한 빨리 새로 고치십시오' 확인란이 선택되어 있으면 예약된 데이터 새로 고침이 작동하지 않습니다.

SQL Server 2012 SharePoint용 PowerPivot에서는 데이터 새로 고침 않습니다.
SQL Server 2012i;"> 관리 일정에 "가능한 빨리 새로 고치십시오" 확인란이 선택된 통합 문서의 새로 고침을 건너뜁니다. 사용자가 데이터 새로 고침 관리 일정에서 확인 단추를 클릭하는 즉시 SharePoint용 PowerPivot에서는 통합 문서를 새로 고치지만 모든 후속 새로 고침 주기는 건너뜁니다. 첫 번째 새로 고침 이후 통합 문서는 더 이상 새로 고쳐지지 않습니다. 데이터 새로 고침 기록에는 데이터 새로 고침 시도가 더 이상 표시되지 않습니다.

2.2 PowerPivot 관리 대시보드의 거품형 차트 Control이 작동하지 않을 수 있음

PowerPivot 관리 대시보드에서 SharePoint 서버의 국가별 설정이 다음과 같이 되어있다면 거품형 차트 control이 작동하지 않습니다.

· 영향을 받는 형식(버그가 나타남): 일/월/년, 년/일/월(예: 유럽 언어, 인도어 등)

· 영향을 받지 않는 형식(버그 없음): 월/일/년, 년/월/일(예: 미국, 일본 등)

2.3 SharePoint용 PowerPivot을 사용하여 .bism 파일을 만들면 유효성 검사에 실패할 수 있습니다.

BI 의미 체계 모델 연결 파일(.bism)은 유효성이 검사되는 시점과 사용되는 시점 사이에 서로 다른 보안 환경에서 실행됩니다. 사용하려면 응용 프로그램의 서비스 계정이 대상 서버의 SSAS 서버 관리자여야 합니다. 그러나 유효성을 검사하는 동안 SharePoint 웹 응용 프로그램의 서비스 계정이 사용됩니다. 따라서 .bism 파일을 만들고 있는 사용자가 유효성 검사 권한이 있는 웹 있는 사용자가 유효성 검사 권한이 응용 프로그램에 없을 수 있지만 파일을 실제로 사용하면 잘 작동합니다. 이러한 경우인지 확인하려면 유효성 검사에 실패할 때 "연결 유효성 검사 없이 링크 파일 저장" 상자를 선택하고 확인을 누르기만 하면 됩니다. .bism 파일을 사용할 응용 프로그램(예: Excel Services 또는 Reporting Services)에 필요한 권한이 있는지 확인한 다음(즉, 해당 서비스 계정이 대상 서버의 다음( SSAS 서버 관리자인지 확인) 해당 파일을 사용해 보십시오. 유효성 검사에 실패했더라도 .bism 파일이 실제로는 작동할 수 있습니다.
