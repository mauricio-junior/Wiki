Merhabalar.. Bu makalemizde, Guest  Object For Active Directory yazılımını kullanarak, Windows Server 2008 Active Directory Domain ‘i içerisinden silinmiş olan bir kullanıcının veya objenin, geri getirilmesi (Restore) işlemlerinden bahsediyor olacağız. Uygulama tarafına geçmeden önce, pogramın kullanılabilirliği konusunda kısaca bilgi vererek makaleme devam etmek istiyorum.

Object Restore For Active Directory

Object Restore For Active Directory, Windows Server 2003 ve Windows Server 2008 Active Directory Domain ortamında bulunan, silinmiş nesnelerin geri getirilmesi işlemlerinde kullanılan ücretiz bir araç olarak karşımıza çıkmakta.  Bu araç sayesinde, Active Directory Domain’i içerisinde, yalnışlıkla silmiş olduğumuz objeleri geri getirebiliyoruz.

Ürünü http://www.quest.com/object-restore-for-active-directory/ adresinden download edebilirsiniz. Ürünün kurulumu gayet basit olduğu için kurulum aşamalarına değinmeyeceğim. Evet, ürün hakkında da kısaca bilgi verdikten sonra, uygulama kısmına geçiş yapıyor olalım. Ben Restore işlemlerini Windows Server 2008 R2 Enterprise işletim sistemi ile çalışan Active Directory Domain rolüne sahip sunucu üzerinde gerçekleştiriyor olacağım. Test işlemi için ben daha önceden Active Directory Domain’i içerisinde Usersklasörü altında yeni bir User (Kullanıcı) oluşturmuştum. Aşağıdaki şekil üzerinde de bunun örneğini görebilirsiniz.

İşlemimize Active Drectory Domain ‘i içerisinde oluşturmuş olduğum “yavuz” isimli kullanıcıyı silerek başlıyoruz.

Yukarıdaki şekil üzerinde de görmüş olduğunuz gibi, Active Directory içerisinden “yavuz” isimli kullanıcıyı silmek isteyip istemediğimiz bize sorulmakta. Biz de YES  tuşuna basarak ilgili kullanıcının silinmesi işlemini onaylıyoruz. Active Directory Domini içerisinden silmiş olduğumuz “yavuz” isimli kullanıcının Object Restore For Active Directory konsolü üzerinde de yer aldığını görmekteyiz.

Active Directory içerisinden silmiş olduğumuz kullanıcıyı geri getirmek için ise, konsol üzerinde silinmiş olan kullanıcının üzerine gelip sağ tuş yapıyoruz ve ardından Restore seçeneğiy konsolü üzerinde de yer aldığını görmekteyiz.

Restore işleminin ardından, yukarıdaki şekil üzerinde de görüldüğü gibi, silinmiş objenin başarılı bir şekilde geri getirilmiş olduğu mesajını alıyoruz. Geri getirme işleminin sağlıklı bir şekilde gerçekleşip gerçekleşmediğini test etmek için ise, tekrar Active Directory Users And Computers ekranına geçiş yapıyoruz. Bakalım restore işlemi gerçekten de başarılı bir şekilde tamamlanmışmıJ

Active Directory And Computers ekranına  baktığımızda ise , Restore işleminin başarılı bir şekilde tamamlanmış olduğunu görüyoruz.  Geri getirmiş olduğumuz kullanıcı ile tekrar sunucumuz üzerine Loginolmak istediğimiz de ise , aşağıdaki gibi bir hata almamız muhtemel.

Yukarıdaki hatayı almamızın sebebi ise, Active Directory içerisinden silinen bir kullanıcı, tekrar geri getirildği zaman hesap otomatik olarak devre dışı bırakılmaktadır. Şekil üzerinde de Login işlemi gerçeklekleştirmek istediğimiz hesabın disable konumunda olduğu bize bildirilmekte. Hesabı enable durumuna getirmek için ise, tekrar Active Directory Users And Computers ekranına dönüş yapıyoruz.

Active Directory Users And Computers ekranı üzerinde ise,kullanıcı hesabını enable yapmak içi,  geri getirmiş olduğumuz kullanıcı üzerine gelip  sağ tuş yapıyoruz ve ardından açılan menü içerisinden Enable Account seçeneğine tıklıyoruz.

Enable Account seçeneğinin de ardıdan, yukarıdaki şekil üzerinde de görmüş olduğunuz gibi bit hata penceresi ile karşılaşıyoruz. Bu hatayı almamızın sebebi de, geri getirmiş olduğumuz kullanıcı üzerinde yeni bir password oluşturmamış olduğumuzdandır. Bu nedenle, yine kullanıcımız üzerinde sağ tuş yaparak, gelen menü içerisinden Reset Password tuşu yardımı ile yeni bir password oluşturuyoruz.

Rest Password ekranında ise, şifremizi yazdıktan sonra, User must change password at next logonseçeneğindeki işareti kaldırıyoruz ve OK diyoruz.

Password oluşturma işlemimiz tamamlandı. Şimdi ise, tekrar yavuz isimli kullanıcımız üzerinde sağ tuş yaparak kullanıcımızı enable yapmayı deniyoruz.

Şekilde de görüldüğü gibi, kullanıcı hesabımızı başarılı bir şekilde enable durumuna getirmiş bulunuyoruz. Bu işlemden sonra, tekrar yavuz isimli kullanıcımız ile Active Directory sunucumuza Login olmayı deniyoruz.

Kullanıcı adı ve password’ü yazıp Enter diyoruz.

Ve Masaüstü ekranımız..
