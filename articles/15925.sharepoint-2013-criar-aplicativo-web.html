Objetivo

O objetivo deste artigo é ensinar como realizar o processo de criação de um aplicativo web no SharePoint Server 2013.

Visão Geral

Para que o SharePoint 2013 consiga criar a estrutura de conteúdo de um farm, faz-se necessário que a solução gere um objeto chamado de aplicativo web. Esta web application na versão inglesa é responsável por armazenar as informações dos elementos nele disposto. O aplicativo web nada mais é que uma coleção de registro de informações de acesso ou endereços gerenciado pelo centro de controle do Microsoft SharePoint 2013. Essas informações são destinadas a replicar todos esses dados no IIS – Internet Information Service de cada servidor de sua estrutura física. Assim, esta configuração do farm é totalmente flexível, o que permite utilizar os aplicativos web em diversos cenários, que vão desde os mais mais simples aos mais complexos e críticos.

Referência Microsoft: http://technet.microsoft.com/en-us/library/cc261875.aspx

Os aplicativos web são mais utilizados na estruturação da hierarquia de sites.

Será solicitado a inserção das informações de configuração do farm, a qual podemos destacar as seguintes:
  • Padronize o nome o aplicativo web que será criado para cada serviço. 
  • Desta forma, o processo indicado além de facilitar o gerenciamento ajudará também em sua manutenção. 
  • Esta configuração deve ser definida em Criar um novo site no IIS.
  • A porta padrão de acesso é a 80 que oferece suporte a Internet, utilize esta porta para definir um local central para acesso. 
  • Além disso, outras portas também poderão ser utilizadas por outros aplicativos web que serão criados no futuro.
