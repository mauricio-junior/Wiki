Introdução



Visão Geral:

  • O Configuration Wizard é o assistente de instalação do SharePoint que auxilia o usuário durante a implantação do produto.
 
INICIANDO:

  • Durante o processo de instalação de um novo ambiente do Sharepoint executamos o "Sharepoint Products Configuration Wizard" e na opção "Specify Configuration Database Setting" utilizamos essa configuração para informar os dados de acesso para criação dos bancos e validação dos usuários de serviço.
Exemplo:



Ao clicar em: Next - "Avançar" após informar os dados é exibido o seguinte erro:

"Cannot connect to database master at SQL Server at servername.yourdomain.org. The database might not exist, or the current user does not have permission to connect to it".

  

Importante:

  • Esse erro está relacionado a alguma informação incorreta, falta de permissão ou problemas com os usuários de serviço do domínio.


 Resolução:



  1. Verifique as permissões no SQL Server  para o usuário que está sendo utilizado no processo de instalação.

    • As permissões padrões são (dbcreator e securityadmin) conforme referência Microsoft: http://technet.microsoft.com/pt-br/library/cc825317.aspx
    • É importante também ter permissão de administrador local para que seja possível rodar o Configuration Wizard sem restrições.

  2. Teste a conectividade via SQL Management Studio utilizando diretamente o servidor SQL Server.
    • Salve um arquivo do bloco de notas com a extensão *.udl em um diretorio. Exemplo: teste.udl, depois de salvar feche o bloco de notas e acesse o arquivo, nele é possível informar os dados de conexão de acesso a instância do SQL Server.
  1. Valide as regras de firewall do Sharepoint e veja se consegue se conectar na pota 1433.
    • Geralmente o que costuma interferir nesses pontos são o firewall do servidor ou da rede, verifique esses pontos.
