1.0 Antes da instalação

Antes da instalação do PowerPivot, leia as seguintes ações necessárias.

1.1 Ocorrerá falha na instalação do PowerPivot se o VSTO 4.0 Runtime não estiver instalado

Você deve instalar o Visual Studio Tools for Office (VSTO) 4.0 Runtime antes de instalar o PowerPivot. Você pode baixar o VSTO Runtime na página do Centro de Download da Microsoft . Este problema ocorre apenas para usuários do Office 2010. Quando o Office 2010 SP1 for lançado, não será necessário fazer um download separado do VSTO Runtime.

1.2 O .NET Framework 4.0 precisa estar instalado no computador

Se você não tiver o .NET Framework 4.0 instalado no computador, o SQL Server 2012, versão do PowerPivot para Excel, será instalado com êxito, mas a guia PowerPivot não aparecerá na faixa de opções do Excel. Você deve instalar o .NET Framework 4.0.

1.4 O PowerPivot para SharePoint requer o SharePoint Server 2010 SP1

Se você pretende usar os recursos do SQL Server 2012 em um ambiente do SharePoint, o SharePoint Server 2010 Service Pack 1 (SP1) será necessário. É também recomendável instalar as correções mais recentes para o SharePoint Server 2010 SP1.

Solução alternativa: você deve instalar o SP1 e corrigir completamente o farm de servidores antes de adicionar os recursos do SQL Server ao farm. Este requisito aplica-se aos seguintes recursos do SQL Server 2012:

· Usar uma instância do Mecanismo de Banco de Dados como o servidor de banco de dados do farm

· Configurando o PowerPivot para SharePoint

· Implantando o Reporting Services no modo do SharePoint.

1.5 Pastas de trabalho PowerPivot para SharePoint anteriores não são atualizadas automaticamente com o SQL Server 2012

Quando você atualiza sua instalação existente do PowerPivot para SharePoint para o SQL Server 2012, versão do PowerPivot para SharePoint, suas pastas de trabalho PowerPivot para SharePoint existentes não são atualizadas automaticamente por padrão. Sem uma atualização, as pastas de trabalho poderão ser usadas no SharePoint, mas o recurso de Atualização de Dados não obterá êxito para pastas de trabalho do SQL Server 2008 R2 PowerPivot. As pastas de trabalho do SQL Server 2012 PowerPivot não são afetadas. No entanto, você pode configurar um recurso de atualização automática em uma instância do PowerPivot para SharePoint que atualiza pastas de trabalho do SQL Server 2008 R2 com a versão mais recente, habilitando a atualização de dados para essas pastas de trabalho. Para obter mais informações, consulte Atualizar PowerPivot para Excel e Dados do PowerPivot.

Solução alternativa: se você não quiser habilitar a atualização automática das pastas de trabalho PowerPivot, deverá baixar as pastas de trabalho SharePoint, atualizá-las manualmente abrindo-as no Excel e, em seguida, carregá-las novamente no SharePoint. A pasta de trabalho em uso não poderá ser atualizada; você também deverá atualizá-la manualmente.

2.0 Problemas conhecidos

Os seguintes problemas conhecidos aplicam-se ao Microsoft SQL Server 2012 PowerPivot para SharePoint.

2.1 Atualização de Dados Agendada não funciona quando 'Também atualizar o mais rápido possível' está marcado

O SQL Server 2012 PowerPivot para SharePoint ignora a atualização de pastas de trabalho que têm a caixa “Também atualizar o mais rápido possível” habilitada na agenda de atualização de dados. O PowerPivot para SharePoint atualizará a pasta de trabalho assim que o usuário clicar no botão OK na agenda de atualização de dados, mas ignorará todos os ciclos de atualização subsequentes. Depois da primeira atualização, a pasta de trabalho não será mais atualizada. O histórico de atualização de dados não mostrará as futuras tentativas de atualizar os dados.

2.2 O controle de gráfico de bolhas do Painel de Gerenciamento PowerPivot não funciona

No Painel de Gerenciamento PowerPivot, o controle de gráfico de bolhas não funciona quando as configurações regionais do SharePoint Server estão especificadas da seguinte forma:

· Formato afetado (ocorre um bug): dia/mês/ano, ano/dia/mês (exemplos: idiomas europeus, indiano, etc.)

· Formato não afetado (não ocorre bug): mês/dia/ano, ano/mês/dia (exemplos: inglês americano, japonês, etc.)

2.3 Pode haver falha na validação ao criar um arquivo .bism usando o PowerPivot para SharePoint

Um arquivo de conexão de modelo semântico BI (.bism) é executado em diferentes ambientes de segurança entre quando é validado e quando é usado. Para uso, a conta de serviço do aplicativo deve ser um administrador do SSAS Server para o servidor de destino. Mas, durante a validação, a conta de serviço do aplicativo Web do SharePoint é usada. Portanto, não é incomum estar em uma situação em que o usuário que está criando o arquivo .bism pode não estar em um aplicativo Web que tem permissões para validar, mas, quando o arquivo é realmente usado, ele é bem-sucedido. Para ver se esse é o caso, quando a validação falhar, marque a caixa "Salvar o arquivo de vínculo sem validação da conexão" e clique em OK. Verifique se o aplicativo que usará o arquivo .bism, por exemplo, Serviços do Excel ou Reporting Services, tem as permissões necessárias (ou seja, que sua conta de serviço seja um administrador do servidor SSAS no servidor de destino) e, em seguida, tente usar o arquivo. O arquivo .bism poderá realmente funcionar mesmo se a validação tiver falhado.
