Resultado:

 

 

Asegúrese de tener la misma dirección IP estática asignada a su máquina virtual de SAP en la biblioteca de SAP Cloud Appliance y en Azure Portal.

 

  1.  Genere una segunda dirección IP en su NIC y asigne la dirección IP estática correcta.
  2.  Generar un "NIC temporal" temporal
  3. A continuación, disociar el "NIC2-original" de la VM,
  4. Ahora haga que la segunda dirección IP sea primaria para que pueda asegurarse de tener asociada la dirección IP correcta.
Si necesita cambiar / actualizar la dirección IP en Azure Portal, puede hacerlo a través de Azure Shell con el siguiente cmd:

 

$ az network nic ip-config update --name "SAP-PublicIP" --nic-name "SAP-Nic2" --resource-group "SAPCAL-Network-westeurope" --make-primary
 
  ** Asegúrate de detener tu VM antes de realizar este proceso

 

 

 

Ahora vuelva al original de NIC2 y asócielo a la máquina virtual:

 

 

Ahora desasocia y elimina el Nic-temporal.
