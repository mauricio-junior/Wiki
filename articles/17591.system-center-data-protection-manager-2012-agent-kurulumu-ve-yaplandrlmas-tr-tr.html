Merhaba arkadaşlar bu makalemizde SCDPM 2012 yönetimi için Client’ larımıza Agent yükleme işlemini gerçekleştireceğiz. Bildiğiniz üzere SCDPM’ de yedekleme işlemi yapabilmemiz için agent kurulumu yapmamız gerekir. Bu kurduğumuz Agent’ın amacı, yüklü olduğu client'ların üzerleinde bulunan Roller, Featurelar ve yazılım hakkında bilgi toplayarak. SCDPM Sunucumuza bu verileri ileterek client'larımızın üzerinde işlem yapma olanağını bizlere sunar.

Agent ile DPM sunucumuz arasında veri iletişiminin gerçekleştirilmesi için DPM Sunucumuzda ve Agent yüklenecek client’ımızda açılması gereken belirli portlarımızı iletişim için açmamız gerekir.

Details

DCOM

135/TCP

The DPM control protocol uses DCOM. DPM issues commands to the protection agent by invoking DCOM calls on the agent. The protection agent responds by invoking DCOM calls on the DPM server.

Dynamic

TCP port 135 is the DCE endpoint resolution point used by DCOM.
 

By default, DCOM assigns ports dynamically from the TCP port range of 1024 through 65535. However, you can configure this range by using Component Services. For more information, see Using Distributed COM with Firewalls (http://go.microsoft.com/fwlink/?LinkId=46088).
 

Note that for DPM-Agent communication you must open the upper ports 1024-65535. To open the ports, perform the following steps:
 

  
 

1. In IIS 7.0 Manager, in the Connections pane, click the server-level node in the tree.
 

  
 

2. Double-click the FTP Firewall Support icon in the list of features.
 

 

  
 

3. Enter a range of values for the Data Channel Port Range.
 

  
 

4. After you enter the port range for your FTP service, in the Actions pane, click Apply to save your configuration settings.

Artık Client’ ımıza agent kurulumumuzu gerçekleştirelim.

DPM Administrator Consolumuzu açtıktan sonra sol altta bulunan Management‘ a tıkladıktan sonra Sol bar’ da açılan menüden Agents‘ a tıklıyoruz. Yukarıda bulunan Install butonuna tıklıyoruz.

Karşımıza gelen Agent Deployment Method penceresinde bulunan ilk seçenek Install agent : Domain’ e dahil Sunucu veya client’ lara ilk defa yüklenecekse işaretlenir. Attach agents : Domain yapısı olmayan ortamlarda veya domain’ e dahil olmayan Sunucu ve Client’ lara manual olarak kurulmuş agent bulunan Sunucu/Clientları DPM konsolumuza göstermek için seçeriz. Biz ilk kez kuracağımız ve domain’ e dahil bir sunucu olduğundan Install agents‘ ı işaretliyoruz ve Next ile devam ediyoruz.

Select Computers penceresinde domain’ imizde bulunan sunucuları görüyoruz. AD Sunucumuza agent yükleyeceğiz. O yüzden Computer Name kısmındayken AD Sunucumuzu seçtik ve Add diyerek Selected Computers kısmına taşıdık. Next ile devam ediyoruz.

Enter Credentials penceresinde agent yükleme işlemini gerçekleştirecek olan yetkili kullanıcı bilgilerini giriyoruz. Eğer bu sunucumuz domain ortamındaysa Domain Admin kullanıcınızı, domaine dahil değilse o sunucunun Local Admin hesabını giriyoruz.

Choose Restart Method penceresinde Agent yüklenen sunucumuzun yükleme işleminden sonra yeniden başlatılıp, başlatılmayacağını belirliyoruz. Ben yeniden başlatılmasını istiyorum ve Yes‘ i işaretliyorum ve Next ile devam ediyoruz.

Summary penceresinde agent yükleme işlemi için hazır olduğunu görüyoruz. Install diyerek kurulum işlemini başlatıyoruz.

Installation penceresinde agent yükleme işlemimizin başarılı bir şekilde yüklendiğini görüyoruz. Bu pencereyi 

Installation penceresinde Close diyerek kapatıyoruz.

Evet arkadaşlar Agent yükleme işlemimizi başarılı bir şekilde tamamladık. Yükleme işleminden sonra agent yüklenen sunucumuzun Programs and Features konsolumuzdan agent’ ımızın yüklendiğini görebilirsiniz.

Artık SCDPM 2012 Sunucumuz Agent yüklü olan sunucumuzun üzerinde bulunan Rolleri, Featureları ve Yazılımlar hakkında bilgileri toplayarak DPM sunucumuza iletecek ve Backup işlemi yapacağımız zaman Client Sunucumuzun içeriğini bize tamamiyle gösterecek ve istediğimiz backup’ ı almamızı sağlayacaktır.
