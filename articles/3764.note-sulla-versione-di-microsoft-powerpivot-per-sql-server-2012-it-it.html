1.0 Operazioni preliminari all'installazione

Prima di installare PowerPivot, leggere le azioni necessarie indicate di seguito.

1.1 L'installazione di PowerPivot non riesce se il runtime di VSTO 4.0 non è installato.

È necessario installare il runtime di Visual Studio Tools per Office (VSTO) 4.0 prima di installare PowerPivot. È possibile scaricare il runtime di VSTO da questa pagina dell'Area download Microsoft . Questo problema si verifica solo per gli utenti di Office 2010. Se viene rilasciato Office 2010 SP1, non è necessario un download distinto del runtime di VSTO.

Area download Microsoft . Questo problema si verifica solo per gli utenti di Office 2010. Se viene rilasciato Office 2010 SP1, non è necessario un download distinto del runtime di VSTO.

1.2 È necessario che sul computer in uso sia installato .NET Framework 4.0

Se sul computer in uso non è installato .NET Framework 4.0, la versione di PowerPivot per Excel di SQL Server 2012 viene installata correttamente, ma la scheda PowerPivot non viene visualizzata nella barra multifunzione di Excel. È necessario installare .NET Framework 4.0.

1.3 È necessario che sul computer in uso sia installato Silverlight 5

In PowerPivot per SharePoint e Power View sono inclusi componenti basati su Silverlight per i quali è richiesta la disponibilità di Silverlight 5 sul computer in uso. Se non si dispone della versione richiesta di Silverlight, sarà necessario installarla quando si passa a un'area dell'applicazione Web in cui è necessaria. Silverlight 5 può essere installato da http://go.microsoft.com/fwlink/?LinkId=243096.

1.4 Per poter utilizzare PowerPivot per SharePoint è necessario aver installato SharePoint Server 2010 SP1

Se si prevede di utilizzare le funzionalità di SQL Server 2012 in un ambiente SharePoint, è necessario disporre di SharePoint Server 2010 Service Pack 1 (SP1). È consigliabile inoltre installare le patch più recenti per SharePoint Server 2010 SP1.

Soluzione alternativa: è necessario installare SP1 e applicare patch alla server farm prima di aggiungere le funzionalità di SQL Server alla farm. Questo requisito si applica alle funzionalità di SQL Server 2012 seguenti:

· Utilizzo di un'istanza del motore di database come server di database della farm

· Configurazione di PowerPivot per SharePoint

· Distribuzione di Reporting Services in modalità SharePoint.

1.5 Le cartelle di lavoro precedenti di PowerPivot per SharePoint non vengono aggiornate automaticamente con SQL Server 2012

Quando si aggiorna l'installazione di PowerPivot per SharePoint esistente alla versione di PowerPivot per SharePoint di SQL Server 2012, le cartelle di lavoro di PowerPivot per SharePoint esistenti non vengono aggiornate automaticamente. Se non si esegue l'aggiornamento, è possibile utilizzare le cartelle di lavoro in SharePoint, tuttavia Aggiornamento dati non verrà eseguito correttamente per le cartelle di lavoro di SQL Server 2008 R2 PowerPivot. Le cartelle di lavoro di SQL Server 2012 PowerPivot non sono interessate. Tuttavia, è possibile configurare una funzionalità di aggiornamento automatico in un'istanza di PowerPivot per SharePoint tramite cui le cartelle di lavoro di SQL Server 2008 R2 vengono aggiornate alla versione più recente, abilitando in questo modo Aggiornamento dati per queste cartelle di lavoro. Per ulteriori informazioni, vedere Aggiornare PowerPivot per Excel e dati PowerPivot.

Soluzione alternativa: se non si desidera abilitare l'aggiornamento automatico delle cartelle di lavoro di PowerPivot, è necessario scaricare le cartelle di lavoro da SharePoint, aggiornarle manualmente aprendole in Excel, quindi caricarle nuovamente in SharePoint. La cartella di lavoro in uso non verrà aggiornata e sarà necessario effettuare questa operazione comunque manualmente.

2.0 Problemi noti

I problemi noti riportati di seguito si applicano a Microsoft SQL Server 2012 PowerPivot per SharePoint.

2.1 L'aggiornamento dei dati pianificato non funziona se è selezionata l'opzione "Aggiorna anche appena possibile"

Con SQL Server 2012 PowerPivot per SharePoint viene ignorato l'aggiornamento delle cartelle di lavoro per cui è abilitata la casella di controllo "Aggiorna anche appena possibile" nella pianificazione dell'aggiornamento dati. Con PowerPivot per SharePoint, l'aggiornamento della cartella di lavoro verrà effettuato non appena l'utente fa clic sul pulsante OK nella pianificazione dell'aggiornamento dati. Tuttavia, tutti i cicli di aggiornamento successivi verranno ignorati. Dopo il primo aggiornamento, la cartella di lavoro non viene più aggiornata. Nella cronologia dell'aggiornamento dati non verranno visualizzati ulteriori tentativi di aggiornamento dati.

2.2 È possibile che il controllo del grafico a bolle del dashboard di gestione PowerPivot non funzioni

Nel dashboard di gestione PowerPivot il controllo del grafico a bolle non funziona se le impostazioni internazionali del server SharePoint sono specifiche a quanto indicato di seguito:

·2.2 È possibile che il controllo del grafico a bolle del dashboard di gestione PowerPivot non funzioni

Nel dashboard di gestione PowerPivot il controllo del grafico a bolle non funziona se le impostazioni internazionali del server SharePoint sono specifiche a quanto indicato di seguito:

Formato interessato (viene visualizzato un bug): giorno/mese/anno, anno/giorno/mese. Ad esempio: lingue europee, indiano e così via.

· Formato non interessato (nessun bug): mese/giorno/anno, anno/mese/giorno. Ad esempio: inglese (Stati Uniti), giapponese e così via.

2.3 La convalida potrebbe non essere completata correttamente quando si crea un file con estensione bism tramite PowerPivot per SharePoint

Un file di connessione BI Semantic Model (con estensione bism) viene eseguito in diversi ambienti di sicurezza nel periodo compreso tra la convalida e l'utilizzo. Per poter utilizzarlo, l'account di servizio dell'applicazione deve essere un amministratore del server SSAS per il server di destinazione. Tuttavia, durante la convalida, è necessario utilizzare l'account di servizio dell'applicazione Web di SharePoint. Pertanto, potrebbe presentarsi una situazione in cui l'utente che sta creando il file con estensione bism potrebbe non trovarsi in un'applicazione Web per cui dispone delle autorizzazioni di convalida, mentre questa condizione si verifica quando il file viene effettivamente utilizzato. Per verificare l'eventuale occorrenza di questo caso, quando la convalida non viene completata correttamente, selezionare semplicemente la casella di controllo "Salvataggio del file di collegamento senza convalida della connessione", quindi fare clic su OK. Assicurarsi che all'applicazione in cui verrà utilizzato il file con estensione bism, ad esempio Excel Services o Reporting Services, non siano associate le autorizzazioni richieste, cioè che il relativo account di servizio sia un amministratore del server SSAS nel server di destinazione, quindi provare a utilizzare il file. Il file con estensione bism potrebbe effettivamente funzionare anche se la convalida non è stata completata correttamente.
