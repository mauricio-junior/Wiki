Introdução

Para a instalação do Configurations Manager 2012 tenho uma maquina virtual Hm01 que é o controlador de domínio, um servidor Sql Server 2008 R2 chamado hm04 e o servidor Hm07 onde será instalado o Configuration Manager 2012. Todos os servidores estão utilizando Windows Server 2008 R2 Service Pack 1.



Pre-Requisitos de Infraestrutura Para Instalação Configuration Manager

Preparação do Active Directory

Para a instalação do Configuration Manager é preciso expandir o Active Directory Schema e criar o container para a publicação dos server roles para a rede interna.

Expansão do Active Directory Schema
Para expandir o schema execute a ferramenta ExtAdSch.exe que esta localizada na mídia de instalação do Configuration Manager na pasta SMSSETUP\BIN\X64
 

Para verificar se o schema foi expandido com sucesso abra o arquivo de log da ferramenta no local c:\ExtADSch.txt .
 

Criação do System Manager Container
Para publicar os componentes do Configuration Manager crie um container na pasta System, para isso inicie o ADSIEdit e conecte-se Default naming context.


Clique com o botão direito na pasta System, selecione New -> Object...


Selecione a classe do objeto igual a container


Configure o valor CN com o nome System Management


E finalize a criação do objeto


Configure o valor CN com o nome System Management


Retorne ao ADSIEdit e abra as propriedades do container criado 


Na aba Security adicione a conta da maquina servidor Configuration Manager na lista de permissão



Configure a conta com Full Controll e clique em Advanced


Edite as permissões da conta


Selecione em Apply to: This Object and all descendant objects e selecione a opção Apply these permissions to objects and/or containers within this container only



Preparação do SQL

As versões de SQL suportadas para instalação do Configuration Manager :
  • SQL Server 2008 SP2 Standard ou Enterprise com Cumulative Update 9 ou superior
  • SQL Server 2008 SP3 Standard ou Enterprise com Cumulative Update 4 ou superior
  • SQL Server 2008 R2 Service Pack 1 com  Cumulative Update 6 ou superior
  • SQL Server Express 2008 R2 com Cumulative Update 4 ou superior (somente para Secondary Sites)
O servidor SQL Server foi habilitado os protocolos de rede para conexão e o serviço iniciado SQL Browser, a collation da instância deve ser configurada SQL_Latin1_General_CP1_CI_AS e suportar autenticação integrada no Windows na instância.



E foi adicionado no grupo administradores locais a conta de maquina do servidor Configuration Manager.



Instalando Pacotes de Pré-Requisitos


Antes de iniciar a instalação dos componentes do Configuration Manager instale as seguintes server features no servidor:
  • .Net Framework 3.5.1
  • WCF Activation
  • .Net Framework 4, download do portal http://www.microsoft.com/download/en/details.aspx?displaylang=en&id=17718 (Instale antes de adicionar o IIS )
  • Background Intelligent Transfer Service (BITS)
  • Remote Diferential Compression

Instale também os seguintes server roles:

  • Windows Server Update Services
  • Web Server IIS
    • Common HTTP Features
      • Static Content 
      • Default Document
      • Directory Browsing
      • HTTP Erros
      • HTTP Redirection
    • Application Development
      • ASP.NET
      • .NET Extensibility
      • ASP
      • ISAPI Extensions
      • ISAPI Filters
    • Health and Diagnostics
      • HTTP Logging
      • Request Monitor
      • Tracing
    • Security
      • Windows Authentication
      • Request Filtering
    • Performance Tools
      • Static Content Compression
      • Dynamic Content Compression
    • IIS Management Tools
      • IIS Management Console
      • IIS 6 Metabase COmpatibility
      • IIS 6 WMI Compatibility
Abra o console de gerenciamento do IIS e configure certificado digital para o binding da porta 443. Este certificado será utilizado para criptografar o trafego do Configuration Manger
 

Com os pre-requisitos cumpridos estamos prontos para inciar a instalação da solução.
