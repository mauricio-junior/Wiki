Sometimes a need arises to stop deploying the current FCS update (hotfix) and revert to deploying the previous release. Each of these releases have been covered in th>

FCS Applicable Hotfixes" Wiki article under the FCS Client Hotfixes section.

In the WSUS console follow these two steps:

  1. Locate the update you want to stop deploying and set it to Not approved. This is important because you don't want to decline the update in the event that the update is revised in the future.
    1. To set an update to Not Approved you must right click on the update, select Approve..., in the Approve Updates dialog right click on your target group(s) and select Not Approved (see graphic below)
  2. Locate the previous update release you want to deploy and set it to Install

 

 

If you have previously set the QFE9 update to Decline you can Search for "2394433" by right clicking on Updates in the left hand pane and choosing Search. When the results show the 2394433 update you can right click on it and select Approve... in order to begin redeploying the update again.
