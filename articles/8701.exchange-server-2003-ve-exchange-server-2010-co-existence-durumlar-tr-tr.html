Bence ilginç bir durumdu

Exchange server 2003 ve exchange server 2010 versiyonlarının co-existence olarak çalıştığı bir exchange organizasyonunda Exchange Server 2003 ‘te açılan/varolan bir mailbox’tan Exchange Server 2010’da açılan bir mailbox’a email gönderilememekteydi. Tam tersi de geçerliydi. Bunun başta routing group connectorler’den kaynaklandığını düşündüm. Kontrol ettim hiçbir sorun yoktu düzgün bir şekilde routing group connector’ler açılmıştı/yapılandırılmıştı. Send ve receive connectoler’de düzgün bir şekilde yapılandırılmıştı.

Google+Technet ikilisinden sonra bunun permission sorunun olduğunu elde ettim.

Çözüm için gerekenler aşağıdaki gibiydi.
1- ADSIEDIT
2- Configuration > Services > MicrosoftExchange > Exchange Organization > Administrative Groups > First Administrative Group > Servers 3- Exchange server 2003 sunucu ismi üzerinde right-click , özellikler
4- Security tab’ı ve advanced butonu
5- “include inheritable permissions” işaretli olup olaadığı kontrol edilip değilse işaretlenmeliydi.
Bu bütün exchange server 2003’ler uzerınde yapılmalıyıdı. Sonuç =Pozitif.
Artık exchange server 2010’daki mailbox-enabled user’lar, 2003’teki mailbox-enabled user’lara email atabilmekteler.
