ISSUE

Non-administrators fail to access the FIM portal. Adding the user to the Administrators set allows access.

process your request.” Configuring detailed error messages in the portal garnered an “Index was out of range. Must be non-negative and less than the size of the collection. Parameter name: index” error.

 

CAUSE

Modifications made to the FIM portal home page removed all the elements from the standard users’ third pane view. At least one home page resource must exist in that pane.

In this case both the “About Forefront Identity Manager” and “Help” resources were removed.

RESOLUTION

Just add back at least one of the resources.

  1. In the FIM portal administrator’s page select “Home Page Resources” from the administration pane.
  2. Select the desired home page resource.
  3. In the general tab, under “Usage Keyword,” add “BasicUI.”
  4. Select OK.
  5. Select Submit.

With one or more resources in the BasicUI standard users may access the FIM portal.

MORE INFORMATION

To enable the detailed error messages please see the “How to Configure Detailed Error Pages for the FIM Portal” wiki entry.
