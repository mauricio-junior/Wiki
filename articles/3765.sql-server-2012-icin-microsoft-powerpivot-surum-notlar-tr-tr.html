1.0 Yüklemeden Önce

PowerPivot'u yüklemeden önce, aşağıdaki gerekli işlemleri okuyun.

1.1 VSTO 4.0 Çalışma Zamanı Yüklü Değilse PowerPivot Yüklemesi Başarısız Olur

PowerPivot'u yüklemeden önce Office için Visual Studio Araçları (VSTO) 4.0 çalışma zamanını yüklemeniz gerekir. VSTO çalışma zamanını bu Microsoft Yükleme Merkezi sayfasından yükleyebilirsiniz. Bu yalnızca Office 2010 kullanıcılarını ilgilendiren bir sorundur. Office 2010 SP1 yayımlandığında, VSTO çalışma zamanı için ayrı bir yükleme gerekli değildir.

1.2 Bilgisayarınıza .NET Framework 4.0 Bulunması Gereklidir

Bilgisayarınızda .NET Framework 4.0 yoksa, Excel için PowerPivot'un SQL Server 2012 sürümü başarıyla yüklenir, ancak Excel şeridinde PowerPivot sekmesi görünmez. .NET Framework 4.0 sürümünü yüklemeniz gerekir.

1.3 Bilgisayarınızda .NET Silverlight 5 Bulunmasını Gerektirir

Hem SharePoint için PowerPivot hem Power View, kullanıcının bilgisayarında Silverlight 5'in yüklü olmasını gerektiren Silverlight tabanlı bileşenler içerir. Silverlight'ın gereken sürümü sizde yoksa, web uygulamasının bu sürümü gerektiren bir alanına gittiğinizde sürümü yüklemeniz istenecektir. Silverlight 5'ı bu adresten yükleyebilirsiniz: http://go.microsoft.com/fwlink/?LinkId=243096.

1.4 SharePoint İçin PowerPivot Ürünü SharePoint Server 2010 SP1'i Gerektirir

SQL Server 2012 özelliklerini bir SharePoint ortamında kullanmayı planlıyorsanız, SharePoint Server 2010 Service Pack 1 (SP1) gereklidir. Ayrıca SharePoint Server 2010 SP1 için en son düzeltme eklerini de yüklemenizi öneririz.

Geçici Çözüm: Sunucu grubuna SQL Server özelliklerini eklemeden önce SP1'i yüklemeniz ve düzeltme eklerini tam olarak gruba uygulamanız gerekir. Bu gereksinim, aşağıdaki SQL Server 2012 özellikleri için geçerlidir:

· Veritabanı Altyapısının bir örneğini grubun veritabanı sunucusu olarak kullanma

· SharePoint için PowerPivot'u Yapılandırma

· Reporting Services'ı SharePoint modunda dağıtma

1.5 Önceki SharePoint İçin PowerPivot Çalışma Kitapları SQL Server 2012 ile Otomatik Olarak Yükseltilmez

Var olan SharePoint için PowerPivot yüklemenizi SharePoint için PowerPivot'un SQL Server 2012 sürümüne yükselttiğinizde, varsayılan olarak var olan SharePoint için PowerPivot çalışma kitaplarınız otomatik olarak yükseltilmez. Yükseltme olmadan çalışma kitapları SharePoint üzerinde kullanılabilir, ancak SQL Server 2008 R2 PowerPivot çalışma kitapları için Veri Yenileme başarısız olacaktır. SQL Server 2012 PowerPivot çalışma kitapları bu durumdan etkilenmez. Ancak bir SharePoint için PowerPivot örneği üzerinde, SQL Server 2008 R2 çalışma kitaplarını son sürüme yükselten otomatik bir yükseltme özelliği yapılandırarak bu çalışma kitapları için Veri Yenileme'yi etkinleştirebilirsiniz. Daha fazla bilgi için bkz. Excel için PowerPivot ve PowerPivot Verilerini Yükseltme.

Geçici Çözüm: PowerPivot Çalışma Kitabı Otomatik yükseltmeyi etkinleştirmek istemezseniz, çalışma kitaplarını SharePoint'ten yüklemeniz, çalışma kitaplarını Excel'de açarak el ile yükseltmeniz ve sonra bunları SharePoint'e geri yüklemeniz gerekir. Kullanım çalışma kitabı da yüklenemez ve bunu da el ile yükseltmeniz gerekir.

2.0 Bilinen Sorunlar

Aşağıdaki bilinen sorunlar, SharePoint için Microsoft SQL Server 2012 PowerPivot için geçerlidir.

2.1 'Ayrıca en kısa zama4 style="margin-top:10pt;margin-right:0in;margin-nda yenile' seçeneği işaretli olduğunda Zamanlanmış Veri Yenileme çalışmaz

SharePoint için SQL Server 2012 PowerPivot, Veri Yenileme zamanlamadan “Ayrıca en kısa zamanda yenile' seçeneği etkin olan çalışma kitaplarını yenilemeyi atlar. SharePoint için PowerPivot, kullanıcı Veri Yenileme zamanlamada Tamam düğmesini tıklar tıklamaz çalışma kitabını yenileyecek, fakat daha sonraki tüm yenileme döngülerini atlayacaktır. İlk yenilemeden sonra çalışma kitabı artık bir daha yenilenmez. Veri Yenileme geçmişinde daha sonraki hiçbir veri yenileme girişimi görünmeyecektir.

2.2 PowerPivot Yönetim Panosu'nun Kabarcık Grafik Denetimi Çalışmayabilir

PowerPivot Yönetim Panosu'nda, SharePoint sunucusunun bölgesel ayarları aşağıdakilere özel olduğunda kabarcık grafik denetimi çalışmaz:

· Etkilenen biçim (hata gösterilir): gün/ay/yıl, yıl/gün/ay (Örnekler: Avrupa Dilleri, Hintçe vb.)

· Etkilenmeyen biçim (hata yok): ay/gün/yıl, yıl/ay/gün (Örnekler: ABD, Japonca vb.)

2.3 SharePoint için PowerPivot kullanılarak bir .bism dosyası oluşturulduğunda doğrulama başarısız olabilir

Bir BI Anlamsal Model bağlantı dosyası (.bism) doğrulandığı ve kullanıldığı zamanlar arasında farklı güvenlik ortamlarında yürütülür. Kullanım için uygulama hizmet hesabı, hedef sunucu için bir SSAS Sunucu yöneticisi olmalıdır. Ancak doğrulama sırasında SharePoint web uygulamasının hizmet hesabı kullanılır. Bu nedenle .bism dosyasını oluşturan kullanıcının doğrulama izinlerine sahip bir web uygulamasında olmadığı ama dosya gerçekte kullanıldığında başarılı olduğu durumlara oldukça sık rastlanır. Bu durumun geçerli olup olmadığını anlamak için, dosya başarısız olduğunda, sadece "Bağlantıyı doğrulamadan bağlantı dosyasını kaydet" kutusunu işaretleyin ve Tamam'ı tıklatın. .bism dosyasının kullanacak olan uygulamanın, örneğin Excel Hizmetleri veya Reporting Services'ın gerekli izinlere sahip olduğundan (yani hizmet hesabının hedef sunucudaki bir SSAS sunucu yöneticisi olduğundan) emin olun ve dosyayı daha sonra kullanmayı deneyin. Doğrulama başarısız olsa bile .bism dosyası çalışabilir.
