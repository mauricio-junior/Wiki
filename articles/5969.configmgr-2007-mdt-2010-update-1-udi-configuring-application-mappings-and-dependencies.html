This article will briefly cover setting up application dependencies and application mappings with UDI (User Driven Installation) with the help of MDT 2010 Update 1 and Configuration Manager 2007 R2. 

 

  • Introduction:   
  • Application dependencies feature in UDI allows on which specific application depends or which applications cannot be installed with/without specific application.  
  • Application mapping helpful to upgrade/replace existing applications. Ex. Office 2007 can be upgraded/replaced with Office 2010  
  • 'Delivery Rule' tab used to deliver/install applications by using specific WMI criteria, like VPN client can be installed on laptops by using WMI query filtered for laptop chassis. 

 

  • Customize the UDI Wizard Configuration File  
  1. Open UDI Wizard Configuration File. [resides in MDT file package with default name UDIWizard_Config.xml ]

      

2.  Click on Application > OSD Applications page  

  • Select desired application
  • Click on 'Edit'
  • Click on 'Dependencies' tab
  • Click on 'Add' and select desired dependency application and select dependency type whether you want to exclude on

   

3.  Click on 'Mappings' tab and specify ADD/Remove display name and MSI product code.

  

4.  Click on 'Delivery Rules' and specify WMI query.

5.  Save xml file with above configurations and close UDI designer window.  

By these configurations, Office 2007 will be replaced by Office 2010 only on Windows 7 PCs during 'PC Refresh'.
