 Windows Server 2008 R2 /w sp1 ( most recent up to date) platformu üzerine Outlook Web Apps Server 2013 ‘ü kurduktan sonra test amaçlı olarak https://webappsserver_fqdn/hosting/discovery url’ini browse ettiğinizde aşağıdaki hatayı alabilirsiniz. 


 

Bu durum .net framework (asp.net ) ’ün register edilmemesinden kaynaklı olabilir. .net framework’ün register edilmiş ve IIS console’unda görünmesi gereken hali aşağıdaki gibi olmalıdır.  



ASP.Net ‘in durumu üstteki gibi görünmüyorsa ilk olarak server manager ’dan Web Server role service’lerinden asp.net ‘in yüklü olup olmadığına bakınız. Yüklü olacaktır. Bu durumda aşağıdaki komut ile asp.net ‘i register etmeniz gerekir ki üstteki gibi IIS Console ’unda görünsün!

Komut isteminde aşağıdaki komutu çalıştırınız.

 %windir%\Microsoft.NET\Framework\v4.0.30319\aspnet_regiis.exe –i

Ardından tekrar https://webappsserver_fqdn/hosting/discovery adresinden test yapabilirsiniz.

Outlook Web Apps Server 2013 ‘un kurulumu ve diğer ürünlerle entegrasyonu için ilgili yazıma bakabilirsiniz. http://www.aydogmusoglu.com/office-web-apps-server-2013-ile-exchange-server-2013-ve-sharepoint-2013-entegrasyonu.html

İyi çalışmalar.
