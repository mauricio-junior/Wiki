Configuramos nos últimos 3 posts o System Center Virtual Machine Manager 2012 R2, e como não tem mais o Self Service Portal (SSP) de instalação junto com o Setup do SVCMM 2012, agora o responsável por isso fica com o System Center App Controller (SCAC).

O App Controller oferece uma experiência universal de autoatendimento que pode ajudá-lo a configurar, implantar e gerenciar máquinas virtuais e serviços entre nuvens privadas e públicas.

Começando a instalação clicaremos em Install e verificar atualizações se precisar. Deixaremos sem serial, pois trabalhamos apenas com versão para testes e estudos.

  

O SCAC confere os pré requisitos e pede como requisito o Console do VMM, então caso você for instalar o SCAC em outra VM, você precisará instalar a console, na imagem abaixo não aparece como requisito pois minha VM já tem instalado. Deixo apenas 2 avisos para mostrar como acontece o aviso sobre a falta de requisitos. Como não temos IIS para gerenciamento via Web ele mostra que o próprio App Controller irá instalar os requisitos faltando, então não precisamos entrar em Server Manager e instalar manualmente.

 

Concluído os requisitos agora poderemos escolher o caminho a ser instalado o App Controller, manteremos o local padrão.

Manteremos a conta de serviço para conta de serviço de rede mesmo, poderíamos adicionar o administrator, mas deixaremos padrão, e porta 18622 também padrão.

No próximo passo ele pede para escolher que configurações queremos para o website, temos a opção All Unassigned ou pelo IP da VM mais a porta 443, e também se irá gerar um certificado ou utilizar um existente. Como já tinha feito certificado coloquei um existente.

Agora irei apontar minha instalação para o SQL2014 que tenho configurado. Apontei para o nome do meu servidor, e decidi colocar também a porta 1433 que é a padrão do SQL. Ao apertar TAB ele já preenche o nome da minha Instância, e já sugere o nome da base de dados como AppController.

A instalação é feita com sucesso e iniciaremos o App Controller pelo Browser.

 Colocando User name e senha, conectamos na Console do App Controller.Iremos conectar nosso SCVMM 2012 R2 clicando em Connect a Virtual Machine Manager server and clouds.

Digitamos um Nome qualquer, mas obrigatório em Connection name, e colocar o nome + FQDN da máquina, e caso for preciso alterar a porta padrão 8100, e deixar ticado para importar o certificado automaticamente.
