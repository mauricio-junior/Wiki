The following procedure describes how to create a new management pack by using the Authoring console and Visual Studio Authoring Extensions.

Using the Authoring Console

To Create a New Management Pack

  1. Start the Authoring console.
  2. Select File, and then select New.
  3. On the Management Pack Template page, in the Select a Management Pack Template box, select Empty Management Pack.
  4. In the Management Pack Identity box, type MyMP.
  5. Click Next.
  6. On the Name and Description page, in the Display Name box, type My Management Pack.
  7. Click Create.
  8. Select File, and then click Save.
  9. Make sure that the File Name box has MyMP.xml, and then click Save.

Using Visual Studio Authoring Extensions

To Create a New Management Pack

  1. Start Visual Studio.
  2. Select File, then New, and then Project.
  3. Select the version of management pack file to create:
    • Select Operations Manager Core Monitoring Management Pack for a management pack that can be used with both Operations Manager 2007 R2 and System Center 2012 Operations Manager.
    • Select Operations Manager 2012 Add-On Management Pack for a management pack that can be used with only System Center 2012 Operations Manager.
  4. In the Name box, type MyMP and click OK.
  5. Configure the Management Group and Deployment options for the management pack:
    1. Right click the MyMP project and select Properties.
    2. Select the Management Group tab.
    3. Click Add.
    4. Type the name of a management server for the management group.
    5. If you require different credentials, then select Log in as a different user.
    6. Click Add Server.
    7. Click the Deployment tab.
    8. Change the Start action to Deploy projects to default management group only.
  6. Select File and then Save MyMP.

To Import an Existing Management Pack

  1. Start Visual Studio.
  2. Select File, then New project from existing Management Pack.
  3. On the Select Management Packs To Convert page, do the following:
    1. Click Add.
    2. Select the management pack file to import and click Open.
    3. In the Platform column, select the version of management pack to create:
      • Select Operations Manager 2007 R2 for a management pack that can be used with both Operations Manager 2007 R2 and System Center 2012 Operations Manager. 
      • Select Operations Manager 2012 for a management pack that can be used with only System Center 2012 Operations Manager. 
    4. Click Next.
  4. On the Provide Paths to Referenced Management Packs page, do the following:
    1. If the Path column for any of the referenced management packs is blank, then click Browse and locate the .mp file.
    2. Click Next.
  5. If you want the management pack to be sealed, then on the Configure a Key File page, do the following: 
    1. Click Browse.
    2. Locate the .snk file with your key pair.
    3. Click Open.
    4. Click Next.
  6. On the Configure Visual Studio Solution page, do the following: 
    1. Select where to import the management pack :
      • To import into the current solution, leave the default selection of Current Solution.
      • To create a new solution, select New Solution and provide a name and path.
    2. Click Next.
  7. On the Convert Management Pack page, do the following: 
    1. Verify that there are no failure messages.
    2. Click Finish.
  8. Configure the Management Group and Deployment options for the management pack:
    1. Righ be sealed, then on the Configure a Key File page, do the following: 
      1. Click Browse.
      2. Locate the .snk file with your key pair.
      3. Click Open.
      4. Click Next.
    2. On the Configure Visual Studio Solution page, do the following: 
      1. Select where to import the management pack :
        • To import into the current solution, leave the default selection of Current Solution.
        • To create a new solt click the MyMP project and select Properties.
        • Select the Management Group tab.
        • Click Add.
        • Type the name of a management server for the management group.
        • If you require different credentials, then select Log in as a different user.
        • Click Add Server.
        • Click the Deployment tab.
        • Change the Start action to Deploy projects to default management group only.
    3. Select File and then Save MyMP.
