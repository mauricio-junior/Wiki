  • The Editor Will Not Saveng you back to the Article page (this is very rare).

    IMPORTANT: Your work is not saved. Do not exit the page.

    Try the following solutions, one at a time:

    1. Click HTML at the bottom of the Editor to go the HTML view.
    2. Scroll through the HTML view and highlight text somewhere in the HTML view.
    3. Click Save below the Editor.
    4. If you go to the Articles page, then your work is saved. If you go back to the Design view of the Editor, then continue to the next step.
    5. Copy the new work you completed.
    6. Exit the Edit tab (for example, click Article above the Editor).
    7. IMPORTANT: Refresh the page. You will see that your new work is not saved.
    8. Click the Edit tab at the top again.
    9. Paste the new work back in the appropriate place, and then format it if you need to.
    10. Click Save.
    11. If the problem persists, try cutting and pasting the content into Notepad to strip all formatting.
    12. Copy the content back into the Editor.
    13. Reformat the content.
    14. Click Save.
    15. If the problem persists, try going back to Step 1.

    For All Table Issues: Create a "Quick and Basic" Table Using the HTML Editor

    TechNet Wiki Editor might not translate your table HTML code the way you want it to, especially if you are pasting from a word processor. To fix this issue, we recommend that you follow this guide to create a quick and basic table that you can fill out with your content.

    See Create Quick and Basic Tables Using the Wiki HTML Editor for information about creating tables using the Wiki HTML editor. This method requires that you edit the raw HTML for your article by clicking the <> HTML option in the status bar at the bottom of the Wiki Editor.
