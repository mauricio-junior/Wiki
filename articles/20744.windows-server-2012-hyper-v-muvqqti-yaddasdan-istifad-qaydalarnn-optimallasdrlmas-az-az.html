Bu günkü məqaləmizdə Hyper-V sistemində VM-ların müvəqqəti yaddaşdam istifadə qaydası və müvəqqəti yaddaşdan istifadənin optimallaşdırılması qaydalarından danışacayıq.

Bilirik ki VM-lar işə düşən zaman daha cox müvəqqqəti yaddaş tələb olunur nəinki VM-lar iləyən zaman. VM-ların açılması prosesini optimallaşdırmaq üçün Hyper-V hypervizorunda  Startup RAM anlayışı var.

Startup RAM – VM –ların işə düşməsi prosesində lazım olan müvəqqəti yaddaşdır.

Dynamic Memory- anlayışına aydınlıq gətirməzdən əvvəl VM-ların müvəqqəti yaddaşdan istifadəsinin optimallaşdırılması qaydasına aydınlıq gətirək. Administrator VM-lara Hyper-V Integration Services yüklədikdən sonra Hypervisor MV-lara Dynamic Memory Virtual Service Consumer driver-ini əlavə edir. Bu driver  hypervisora virtual maşının işləməsi üçün  cari vaxtda nə qədər müvəqqəti yaddaş tələb etdiyini təyin etməyə kömək edir.  Hyper-V Integration Services VM-a syükləmək üçün Action menyusundan insert integration services setup Disk seçilməlidir. Bundan sonra VM da CD-ROM bölməsinə virtual disk əlavə olunacaq. Bu diskin içərisində servisin yükləmə faylı vasitəsilə Hyper-V Integration Services VM lara yüklənir.



Dynamic Memory

Minimum RAM-  Virtual maşının normal işləməsi üçün lazım olan minimum müvəqqəti yaddaşı xarakterizə edir. Tutaq ki Hypervizor oolan serverin üzərində 32000MB fiziki yaddaş mövcuddur. Əgər biz VM-da Müvəqqəti yaddaşı 8000 MB təyin ediriksə bu zaman 8000MB müvəqqəti yaddaş həmin VM üçün hypervisor vasitəsilə rezerv edilir. Yəni bu müvəqqəti yaddaşdan digər VM-lar istifadə edə bilməzlər.

Maximum RAM- Bu özəllik VM- işləyən zaman istifadə edə biləcək maksimum müvəqqəti yaddaşı xarakterizə edir.

Tutaq ki VM –a Minimum RAM: 8000 MB maksimum RAM isə 12000 MB təyin edilib.  Cari vəziyyətdə isə VM işləməsi üçün  10000 MB müvəqqəti yaddaş  tələb edir.  Bu tələbatı yuxarıda qey etdiyimiz kimi Hyper-V Integration Services vasitəsilə  VM dan Hypervozora ötürülür. Hypervizor təyin edir ki VM –a təyin edilmiş Maximum RAM dan 2000 MB istifadəsiz qalıb. Bu zaman Hypervizor 2000 MB müvəqqəti yaddaşı lazım olan digər məqsədlərlə (məsələn digər VM –ların normal işləməsinə,  canlı miqrasiya əməliyyatına və s.  ) istifadə edir.

Memory Buffer: Neçə faiz müvəqqəti yaddaşın bufferə yazıla biləcəyini təyin edir. Bu funksionallığı Windows  əməliyyat sistemində Paging file özəlliyi ilə  də eyniləşdirmək olar. Tutaq ki VM işə düşən zaman  VM-a lazım olan müvəqqəti yaddaş tələbatı  Hypervisor vasitəsilə ödənə bilmir. Bu zaman Memory buffermüvəqqəti olaraq fiziki yaddaşı RAM cache kimi istifadə edir.
