سناریو اول :مهاجرت Hyper-V  از ویندوز سرور 2008R2 به ویندوز سرور 2012

سلام

پس از سری آموزش هایی که برای Hyper-V در ویکی های قبلی قرار دادم ، فکر می کنم زمان ان رسید که کمی وارد بحث عملی شویم و چند مثال از دنیا واقعی را مطرح و سپس به حل آن بپردازیم .

در فروم مایکروسافت خیلی از سوالات Hyper-V مربوط به عملیات مهاجرت از Hyper-V 2.0 به Hyper-V 3.0 می باشد. ما نیز در اینجا قصد داریم همین سناریو را پیاده سازی کنیم. در واقع در این سناریو ما هم حجم کاریمان یا Workload بر روی بستر مجازی است و بدلیل قابلیت های جدید Hyper-V 3.0 میخواهیم به این نسخه مهاجرت کنیم با کمترین زمان قطع دسترسی . 



گام اول : (سرور مقصد)

1-      Hyper-V Server 2012را نصب و پیکر بندی کنید.
نصب Hyper-V بر روی ویندوز سرور 2012
Install Hyper-V Server 2012

فراموش کردم نحوه نصب Hyper-V Server  به صورت Core رو توضیح بدهم. فعلا از این لینک استفاده کنید در اولین فرصت ترجمه می کنم و در اختیار شما قرار می دهم.

2-      یک Virtual Networks همانند همانی که در Hyper-V Server 2008  قرار دارد ایجاد کنید.

گام دوم: (سرور مبداء)

1-      از Hyper-V Server 2008  یک پشتیبان یا همان بکاپ تهیه کنید که احیانا اگر مشکلاتی پیش آمد راه برگشت داشته باشید. در طول مهاجرت مشکلی پیش نمی آید.

2-      تمام ماشین های مجازی را خاموش کنید.

3-      همه snapshots مربوط به ماشین مجازی را بازنگری کنید، اگر snapshot از ماشین مجازی  که اجرا بوده است ،گرفته اید را با استفاده از Delete Snapshot ..." پاک کنید. زیرا در Migration (مهاجرت)  Snapshot هایی را که در حال اجرا گرفته اید پشتیانی نمیشوند.

4-      همه ماشین های مجازی را بازنگری کنید و مطمئن شوید که هیچ ماشین مجازی در حالت Saved قرار نداشته باشد ، اگر هم وجود دارد با استفاده از Delete Saved State آن را پاک کنید. به دلیل اینکه وضعیت Saved نیز در طول مهاجرت پشتیبانی نمیشود .

5-      MAC address مربوط به ماشین های مجازی را در حالت Static قرار دهید. تاماشین مجازی یک ارتباط ثابت بین NIC مجازی و MAC address  خود نگه دارد که باعث میشود حتی بعد از مهاجرت به Hyper-V Server 2012 شما نیاز به پیکرندی مجدد NIC مجازی نداشته باشید.

6-        از ماشین مجازی Export بگیرید ، برای این کار بر روی ماشین مجازی راست کلیک و Export ... را انتخاب کنید سپس یک مکان خارجی و یا یک محل اشتراکی که توسط هر دو سرور دسترسی داشته باشد، انتخاب کنید.

گام سوم : (سرور مقصد)

1-      ماشین مجازی را که export گام سوم : (سرور مقصد)

کرده بودید را Import   کنید. برای این کار از منوی Action بر روی Import Virtual Machine ...   کلیک کنید . دقت داشته باشید در یک مرحله از این ویزارد شما با گزینه هایی مثل "Register", "Restore", or "Copy" روبرو میشوید در اینجا بر روی Restore کلیک کنید.

2-      تنظیمات را بازنگری کنید برای این کار بر روی ماشین مجازی راست کلیک و بر روی Settings ... کلیک کنید . شما اکنون میتوانید از قابلیت های Hyper-V 2012 استفاده کنید ، مثل افزایش پردازنده شما میتوانید تا 64 پردازنده اختصاص دهید.

3-      ماشین مجازی را اجرا کنید و تا بررسی کنید که ماشین مجازی به درستی اجرا شده است.

4-      سپس به ماشین مجازی متصل شوید  Connect ... تا مطمئن شوید که ماشین مجازی به صورت کامل بوت شده است .

5-      به ماشین مجازی لاگین کنید و بررسی کنید که سرویس های ماشین مجازی به درستی کار می کنند.

6-      سپس Hyper-V Integration Services را برای ماشین محازی آپدیت کنید. برای این کار از منوی Action -> Insert Integration Services Setup Disk را انتخاب و سپس نصب کنید.
