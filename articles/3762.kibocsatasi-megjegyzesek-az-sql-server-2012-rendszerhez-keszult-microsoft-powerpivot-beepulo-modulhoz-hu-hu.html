1.0 Telepítés előtt

A PowerPivot beépülő modul telepítése előtt olvassa el az alábbi szükséges lépéseket.

1.1 A PowerPivot telepítése sikertelen lesz, ha nincs telepítve a VSTO 4.0 futtatórendszere.

A PowerPivot beépülő modul telepítése előtt telepíteni kell az Office rendszerhez készült Visual Studio eszközök (VSTO) 4.0 futtatórendszerét. A VSTO futtatórendszere a Microsoft letöltőközpont ezen lapjáról 1.1 A PowerPivot telepítése sikertelen lesz, ha nincs telepítve a VSTO 4.0 futtatórendszere.

A PowerPivot beépülő modul telepítése előtt telepíteni kell az Office rendszerhez készült Visual Studio eszközök (VSTO) 4.0 futtatórendszerét. A VSTO futtatórendszere a tölthető le. Ez a probléma csak az Office 2010 felhasználóit érinti. Az Office 2010 1. szervizcsomagjának megjelenése után nem lesz szükség a VSTO futtatórendszerének külön letöltésére.

1.2 Szükséges a .NET-keretrendszer 4.0 a számítógépen.

Ha nincs telepítve a .NET-keretrendszer 4.0 a számítógépen, az SQL Server 2012 részét képező, az Excel programhoz készült PowerPivot beépülő modul sikeresen telepíthető ugyan, de a PowerPivot lap nem jelenik meg az Excel menüszalagján. Telepíteni kell a .NET-keretrendszer 4.0-s verzióját.

1.3 Szükséges a Silverlight 5 a számítógépen.

Mind a SharePoint rendszerhez készült PowerPivot beépülő modul, mind a Power View tartalmaz olyan Silverlight-alapú összetevőket, amelyek működéséhez szükséges, hogy a Silverlight 5 telepítve legyen a felhasználó számítógépén. Ha nincs telepítve a Silverlight szükséges verziója, megjelenik majd egy felszólítás a telepítésére, amikor a webalkalmazás olyan részére navigál, amely megköveteli azt. A Silverlight 5 a http://go.microsoft.com/fwlink/?LinkId=243096 weblapról telepíthető.

1.4 A SharePoint rendszerhez készült PowerPivot beépülő modulhoz szükség van a SharePoint Server 2010 1. szervizcsomagjára.

Ha SharePoint-környezetben kívánja használni az SQL Server 2012 funkcióit, szükség van a SharePoint Server 2010 1. szervizcsomagjára (SP1). Javasolt emellett a SharePoint Server 2010 1. szervizcsomagjához készült legfrissebb javítások telepítése is.

Megoldás: Telepíteni kell az 1. szervizcsomagot és az összes szükséges javítást a kiszolgálófarmra, mielőtt felveszi az SQL Server funkcióit a farmba. Ez a követelmény az SQL Server 2012 alábbi funkcióira vonatkozik:

· Az adatbázismotor egy példányának használata a farm adatbázis-kiszolgálójaként

· A SharePoint rendszerhez készült PowerPivot beépülő modul konfigurálása

· A Reporting Services rendszerbe állítása SharePoint-módban

Frissítés nélkül a munkafüzetek használhatók ugyan a SharePoint rendszerben, de az adatfrissítés sikertelen lesz az SQL Server 2008 R2 részét képező PowerPivot beépülő modulban mentett munkafüzetek esetén. 2.0 Ismert problémák

Az alábbi ismert problémák léphetnek fel a Microsoft SQL Server 2012 részét képező, a SharePoint rendszerhez készült PowerPivot beépülő modulban.

2.1 Az ütemezett adatfrissítés nem működik, ha be van jelölve az „Egyúttal frissítés a lehető leghamarabb” jelölőnégyzet.

Az SQL Server 2012 részét képező, a SharePoint rendszerhez készült PowerPivot beépülő modul kihagyja azon munkafüzetek frissítését, amelyek adatfrissítési ütemtervében be van jelölve az „Egyúttal frissítés a lehető leghamarabb” jelölőnégyzet. A SharePoint rendszerhez készült PowerPivot beépülő modul frissíti a munkafüzetet, amikor a felhasználó az adatfrissítési ütemtervben látható OK gombra kattint, de kihagyja az összes további frissítési ciklust. Az első frissítés után a munkafüzet többé nem frissül. Az adatfrissítési előzmények között nem látható majd az adatok frissítésére irányuló további kísérlet.

2.2 A PowerPivot kezelési irányítópult Buborékdiagram vezérlője esetleg nem működik.

A PowerPivot kezelési irányítópulton levő Buborékdiagram vezérlő nem működik, ha a SharePoint-kiszolgáló területi beállításai a következőket adják meg:

· Érintett formátumok (a hiba jelentkezik): nap/hónap/év, év/nap/hónap (például európai nyelvek, indiai nyelv stb.)

A PowerPivot kezelési irányítópulton levő Buborékdiagram vezérlő nem működik, ha a SharePoint-kiszolgáló területi beállításai a következőket adják meg:

· Érintett formátumok (a hiba jelentkezik): nap/hónap/év, év/nap/hónap (például európai nyelvek, indiai nyelv stb.)· Nem érintett formátumok (a hiba nem jelentkezik): hónap/nap/év, év/hónap/nap (például amerikai angol, japán nyelv stb.)

2.3 Az érvényesítés esetleg sikertelen lesz, amikor a SharePoint rendszerhez készült PowerPivot beépülő modullal hoznak létre .bism kiterjesztésű fájlt.

Egy BI szemantikaimodell-kapcsolatot tartalmazó fájl (.bism) végrehajtása eltérő biztonsági környezetben történik a fájl érvényesítésekor, illetve használatakor. A fájl használatához az alkalmazás szolgáltatási fiókjának az SSAS-kiszolgáló rendszergazdájának kell lennie a célkiszolgálón. Az érvényesítés során azonban a rendszer a SharePoint-webalkalmazás szolgáltatási fiókját használja. Ezért könnyen előfordulhat, hogy a .bism kiterjesztésű fájlt létrehozó felhasználó nem olyan webalkalmazásban dolgozik, amelynek megfelelő engedélye van az érvényesítésre, a fájl tényleges felhasználása azonban sikeres lesz. Sikertelen érvényesítés esetén ellenőrizheti, hogy ez az eset áll-e fenn, ehhez jelölje be „A hivatkozásfájl mentése a kapcsolat érvényesítése nélkül” jelölőnégyzetet, majd kattintson az OK gombra. Győződjön meg arról, hogy az az alkalmazás, amely felhasználja majd a .bism kiterjesztésű fájlt – például az Excel Services vagy a Reporting Services –, rendelkezik a szükséges engedélyekkel (azaz hogy a szolgáltatási fiókja az SSAS-kiszolgáló rendszergazdája a célkiszolgálón), majd próbálja meg használni a fájlt. A .bism kiterjesztésű fájl esetleg sikertelen érvényesítés esetén is megfelelően működhet.
