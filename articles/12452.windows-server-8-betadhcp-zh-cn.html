在一般的教学中，DHCP服务多用于讲解Windows服务的开篇，因此许多人认为其简单，其实不然，想把DHCP服务管理好也不是件容易的事。微软最新发布的Windows Server 8 Beta服务器版操作系统中，为DHCP服务赋予了很多新功能。本文主要想引领读者对该服务的全貌有一个初步了解，从中可以领略新版服务器操作系统的改观，细节问题后续讨论。

安装好Windows Server 8 Beta后，可以在其上直接实验各种新功能，也可以安装Hyper-V服务，再创建虚拟机在其上实验。

搭建服务器的首要条件是要有固定IP地址，所以首先配置服务器网卡的参数，分配固定IP、子网掩码、默认网关，如果有DNS服务器要指明其地址，然后打开powershell窗口测试配置，如图一所示。
 

图一 测试网络配置

检查了网络配置后，需要为服务器更改名称，以方便管理，如图二所示。

 

 

图二 更改服务器名称

更改服务器名称后需要重启系统才能生效，此时服务器已经准备就绪，可以进行DHCP服务的配置工作了。与Windows 8不同，Windows Server 8 Beta开机登录后进入的并不是Metro风&#26684;的“开始”菜单，而是直接进入桌面环境，并且默认情况下不需要创建用户账户而是用管理员账户登录的。登入桌面环境后，请单击左下角的“服务器管理器”按钮，打开“服务器管理器”开始配置DHCP服务，如图三。

 

图三 服务器管理器

与其前任有着天壤之别，本版的服务器管理器界面更加简洁直观，使人一目了然。通过单击“快速启动”中的“添加角色和功能”可以开启一个向导来引导完成服务器的配置工作，如图四。

 

 

图四 添加角色和功能向导

图三 服务器管理器

与其前任有着天壤之别，本版的服务器管理器界面更加简洁直观，使人一目了然。通过单击“快速启动”中的“添加角色和功能”可以开启一个向导来引导完成服务器的配置工作，如图四。

 

 

图五 安装类型选择

接下来是选择服务器，可以是位于服务器池中的服务器，也可以是虚拟硬盘。从这些用词来看，Windows Server 8已经为云计算做好了充分准备。选择服务器池列表中的本机服务器，然后单击“下一步”，如图六。

 

 

图六 服务器选择

接下来首先是服务器角色选择，在列表中找到DHCP服务器，勾选其边上的复选框会弹出对话框，如图七。

 

 

图七 安装DHCP角色

请注意这里的提示文字，此步骤需要选择是否安装管理工具，可以根据网络实际情况勾选下方的“包括管理工具”复选框，这样在安装DHCP角色的同时会将管理工具一同安装在此服务器上，否则不需要勾选。单击“添加功能”按钮后，再单击“下一步”进入功能选择对话框。如果所安装的角色需要相应的功能配合，那么应在此选择。DHCP服务不需要额外的功能配合，所以单击“下一步”后出现的是DHCP安装提示信息，最后提示需要固定IP地址，以及提前做好网络规划并保存在安全的地方，如图八。

 

 

图八 DHCP简要说明

单击“下一步”后，会要求指定安装源路径，如果安装源不在本地服务器，可以在这里指定网络路径，否则直接单击“安装”按钮开始复制文件并初始配置DHCP服务器，如图九。

 

 

图九 指定安装源

图十 安装结束

请注意此处的提示，虽然已经完成了DHCP角色的安装，但需要配置。再有，新版服务器管理器在安装过程启动以后可以关闭向导而去干别的，这一点改变给管理员带来了极的方便。接下来进行后续配置，在服务器管理器的右上角上有一面小旗子的提示图标，单击该图表会弹出提示对话框，如图十一。

 

 

图十一 后续配置

单击图中红色方框中的链接“完成DHCP配置”，调出“DHCP安装后配置向导”，该向导完成两个安全组——“DHCP管理员”和“DHCP用户”的创建，单击“提交”按钮完成创建和委派权限过程。这是新版DHCP服务器的新增功能，如图十二。

 

 

图十二 安装后配置

创建完成后会要求重启DHCP服务以使改变生效，如图十三。

 

图十三 重启服务

服务器重启后，在开始菜单上会出现DHCP服务器管理器的图标，单击可以启动DHCP管理工具，如图十四。

 

 

图十四 DHCP管理器

从图中可以看到，此版DHCP服务增加了“策略”和“筛选器”两个选项。策略选项为企业合作者提供了灵活的配置策略，便于对合作伙伴、供应商等的管理。而筛选器中增加的“允许”和“拒绝”列表功能，类&#20284;ACL（访问控制列表）。DHCP服务虽然搭建完成了，但在没有添加作用域并激活之前，服务器还不能为客户端分配IP地址，所以首先需要创建作用域，如图十五。

 

 

图十五 创建作用域步骤

1、  在“IPv4”节点单击右键，选择“新建作用域”命令；

2、  输入作用域名称及描述，以便于区分和管理多个作用域；

3、  输入作用域的起止IP地址、子网掩码、地址长度；

4、  如果需要在作用域中排除某段或某几个IP，在此页设置；

5、  默认IP租期是8天，可根据情况更改；

6、  这里选择稍后配置；

7、  注意提示信息，需要为作用域配置选项（如果必要），另外，作用域需要激活。

至此作用域创建完毕，可以在图十六中看到该作用域为非活动状态，需要激活才可以对外工作，图十七展示了激活方法。

 

 

图十六 未激活作用域

 

图十七 激活作用域

激活后的作用域如图十八，至此DHCP服务器已经可以为客户端分配IP地址了。对于大型网络还要根据企业的安全策略、业务流程等，进行更为详尽的配置，本文将不再赘述。
