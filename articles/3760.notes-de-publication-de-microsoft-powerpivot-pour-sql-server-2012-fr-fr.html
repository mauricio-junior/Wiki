1.0 Avant l'installation

Lisez les actions requises répertoriées ci-dessous avant d'installer PowerPivot.

1.1 L'installation de PowerPivot échoue si VSTO 4.0 Runtime n'est pas installé

Vous devez installer Visual Studio Tools for Office (VSTO) 4.0 runtime avant d'installer PowerPivot. Vous pouvez télécharger VSTO Runtime à partir de cette page du Centre de téléchargement Microsoft . Ce problème n'affecte que les utilisateurs Office 2010. Si Office 2010 SP1 est utilisé, il n'est pas nécessaire de télécharger VSTO Runtime séparément.

1.2 .NET Framework 4.0 doit être installé sur votre ordinateur

Si . NET Framework 4.0 n'est pas installé sur votre ordinateur, la version SQL Server 2012 de PowerPivot pour Excel s'installe correctement, mais l'onglet PowerPivot n'apparaît pas dans le Ruban Excel. Vous devez installer .NET Framework 4.0.

1.3 Silverlight 5 doit être installé sur votre ordinateur

PowerPivot pour SharePoint et Power View incluent tous les deux des composants basés sur Silverlight qui nécessitent l'installation de Silverlight 5 sur l'ordinateur de l'utilisateur. Si vous n'avez pas la version requise de Silverlight, vous êtes invité à l'installer lorsque vspan>

Sious accédez à une partie de l'application Web qui l'utilise. Vous pouvez installer Silverlight 5 à partir de la page http://go.microsoft.com/fwlink/?LinkId=243096.

1.4 PowerPivot pour SharePoint requiert SharePoint Server 2010 SP1

Si vous envisagez d'utiliser les fonctionnalités SQL Server 2012 dans un environnement SharePoint, SharePoint Server 2010 Service Pack 1 (SP1) est requis. Nous vous recommandons également d'installer les derniers correctifs logiciels disponibles pour SharePoint Server 2010 SP1.

Solution : vous devez installer SP1 et corriger intégralement la batterie de serveurs avant de lui ajouter les fonctionnalités SQL Server. Cette condition s'applique aux fonctionnalités SQL Server 2012 suivantes :

· Utilisation d'une instance du moteur de base de données comme serveur de base de données de la batterie de serveurs

· Configuration de PowerPivot pour SharePoint

· Déploiement de Reporting Services en mode SharePoint.

1.5 Les anciens classeurs PowerPivot pour SharePoint ne se mettent pas automatiquement à niveau avec SQL Server 2012

Lorsque vous procédez à la mise à niveau de votre installation PowerPivot pour SharePoint vers la version SQL Server 2012 de PowerPivot pour SharePoint, vos classeurs PowerPivot pour SharePoint existants ne sont pas mis à niveau automatiquement par défaut. Même s'ils n'ont pas été mis à niveau, les classeurs peuvent être utilisés sur SharePoint mais l'actualisation des données échouera pour les classeurs PowerPivot SQL Server 2008 R2. Les classeurs PowerPivot SQL Server 2012 ne sont pas concernés. Toutefois, vous pouvez configurer une fonctionnalité de mise à niveau automatique sur une instance de PowerPivot pour SharePoint qui met à niveau des classeurs SQL Server 2008 R2 vers la version la plus récente, permettant ainsi l'actualisation des données pour ces classeurs. Pour plus d'informations, consultez Mettre à niveau PowerPivot pour Excel et des données PowerPivot.

Solution : si vous ne souhaitez pas activer la mise à niveau automatique des classeurs PowerPivot, vous devez télécharger les classeurs à partir de SharePoint, les mettre à niveau manuellement en les ouvrant dans Excel, puis les charger à nouveau sur SharePoint. Le classeur d'utilisation ne pourra pas être actualisé. Vous devrez le mettre à niveau manuellement lui aussi.

2.0 Problèmes connus

Les problèmes connus suivants s'appliquent à Microsoft SQL Server 2012 PowerPivot pour SharePoint.

2.1 L'actualisation des données planifiée ne fonctionne pas lorsque la case « Aussi actualiser dès que possible » est activée

SQL Server 2012 PowerPivot pour SharePoint ignore l'actualisation des classeurs pour lesquels la case « Aussi actualiser dès que possible » est activée dans la planification d'actualisation des données. PowerPivot pour SharePoint actualise le classeur dès que l'utilisateur clique sur le bouton OK dans la planification d'actualisation des données, mais ignore tous les cycles d'actualisation suivants. Une fois la première actualisation effectuée, le classeur n'est plus mis à jour. L'historique d'actualisation des données ne montre aucune autre tentative d'actualisation des données.

2.2 La commande de graphique en bulles du tableau de bord de gestion PowerPivot peut ne pas fonctionner

Dans le tableau de bord de gestion PowerPivot, la commande de graphique en bulles ne fonctionne pas lorsque les paramètres régionaux du serveur SharePoint sont spécifiques aux formats suivants :

· Format affecté (un bogue s'affiche) : jour/mois/année, année/jour/mois (exemples : langues européennes, indien, etc.)

· Format non affecté (aucun bogue) : mois/jour/année, année/mois/jour (exemples : anglais (États-Unis), japonais, etc.)

2.3 La validation peut échouer lors de la création d'un fichier .bism à l'aide de PowerPivot pour SharePoint

Format affecté (un bogue s'affiche) : jour/mois/année, année/jour/mois (exemples : langues européennes, indien, etc.)

· Format non affecté (aucun bogue) : mois/jour/année, année/mois/jour (exemples : anglais (États-Unis), japonais, etc.)

Un fichier de connexion de modèle sémantique BI (.bism) s'exécute dans différents environnements de sécurité entre le moment où il est validé et celui où il est utilisé. Pour pouvoir être utilisé, le compte de service de l'application doit être un administrateur de serveur SSAS pour le serveur cible. Mais pendant la validation, le compte de service de l'application Web SharePoint est utilisé. Par conséquent, il n'est pas rare d'assister à une situation dans laquelle l'utilisateur qui crée le fichier .bism peut ne pas travailler dans une application Web disposant d'autorisations de validation, mais cette opération peut aboutir lorsque le fichier est utilisé. Pour vérifier si c'est le cas, lorsque la validation échoue, activez simplement la case « Enregistrer le fichier de liaison sans valider la connexion » et cliquez sur OK. Assurez-vous que l'application qui va utiliser le fichier .bism, Excel Services ou Reporting Services, par exemple, dispose des autorisations requises (autrement dit que le compte de service est un administrateur de serveur SSAS sur le serveur cible), puis essayez d'utiliser le fichier. Le fichier .bism peut être utilisé même si la validation a échoué.
