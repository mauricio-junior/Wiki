Background:

When you attempt to click the Test Connection button on a data source that is configured to use the SharePoint List data source type, you see an error message that resembles the following:

Microsoft.ReportingServices.Diagnostics.Utilities.InvalidDataSourceCredentialSettingException: The current action cannot be completed. The user data source credentials do not meet the requirements to run this report or shared dataset. Either the user data source credentials are not stored in the report server database, or the user data source is configured not to require credentials but the unattended execution account is not specified.

This behavior will occur only when the following are true for the data source:

  • It is created within SharePoint
  • The connection string is configured with a URL to the local SharePoint site
  • It is configured for Windows Integrated Authentication

This issue only affects the Test Connection option within the data source. Using the data source to create and run reports will not result in the InvalidDataSourceCredentialSettingException.

Workaround:

There is no current workaround for this issue so that the Test Connection option within the data source configuration will work. However, the functionality of the data source when used with a report is not affected and the report should successfully retrieve data from the SharePoint List which has been configured within the data source.

Applies to:

  • Reporting Services Add-in for Microsoft SharePoint 2010
