Problem Description:

Content Indexing of all Databases of a DAG Exchange 2010 Mailbox Server Node is in Failed State

Cause:

Microsoft Exchange Search Indexer is in hung state due to this content indexing of all databases of the particular server is in failed state.

Restart the Microsoft Exchange Search Indexer Service on affected Mailbox Server. In most of the cases while restarting this service, it will go into stopping state. To resolve this kill the process of Microsoft.Exchange.Search.ExSearch.exe from the task manager. Now start the Microsoft Exchange Search Indexer service if not already started.
