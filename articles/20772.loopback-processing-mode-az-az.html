Salam Aleykum Dostlar, təxmini bir aydır ki, məqalə paylaşmıram. Düşündüm nə məqalə yazım, sonda qərara aldım ki, Loopback Processing Mode`u izah edim. Bu məqalə praktiki məqalə deyil. Sadəcə olaraq nəzəri cəhətdən şəkillərlə çalışacağıq ki, Loopback Processing Mode`u başa düşək. Dərs deyərkən də bir çox tələbələrə maraqlı olur ki, işləmə prinsipi necə olur, nədir? Loopback Processing Mode`un nə olduğunu cümlə ilə yazmayacağam, çünki bu məqaləmizin sonunda bunun nə işə yaradığını başa düşəcəyik, İnşAllah. Ümumiyyətlə, bir domain düşünək və burada iki ədəd OU olsun. Birini adlandlass="content-fragment-top fiji-content-fragment-top">
ıraq TERMINAL OU, digərini isə USER OU adlandıraq. TERMINAL OU daxilində ancaq kompüterlər olsun, USER OU daxilində isə ancaq istifadəçilər olsun və bu iki OU`ya müəyyən GPO`lar tətbiq edək. Şərti olaraq mən TERMINAL OU`ya Computer Configuration 2 və User Configuration 2 adlanan GPO`lar tətbiq etmişəm. USER OU`ya isə Computer Configuration 1 və User Configuration 1 adlanan GPO`lar tətbiq etmişəm.Bilirəm indi deyəcəksiniz ki, bu nə cümlədir? Hər şey bir-birinə qarışdı. Heç narahat olmayın, öz əllərimlə sizin üçün bütün bunları şəkil üzərində düzəltmişəm.

Məncə artıq indi daha gözəl aydın oldu. İcazənizlə indi məqaləmizə davam edək. İlk öncə Loopback Processing Mode aktiv olmayan hala baxaq. Təsəvvür edək ki, USER OU`da yerləşən istifadəçi LOGİN olur TERMINAL OU`da yerləşən kompüterə və bu zaman GPO`lar aşağıdakı kimi tətbiq olunur.

Şəkildən də gördüyünüz kimi USER OU`na aid olan User Configuration 1 adlı GPO və TERMINAL OU`ya aid olan Computer Configuration 2 adlı GPO tətbiq edilib. Nümunə üçün düşünək,  biz istəyirik ki, istifadəçi nə zaman TERMINAL OU`da yerləşən Terminal Server`ə daxil olur və bu zaman ancaq TERMINAL OU`ya tətbiq etdiyimiz GPO`lar tətbiq olunsun. Məhz bunun üçün biz Loopback Processing Mode istifadə edirik. Bildiyiniz kimi Loopback Processing Mode 2 cür olur və biz hər bu 2 hala baxacağıq, İnşAllah. İlk öncə Loopback Replace Mode`u akiv edərsək nə baş verər? Bunu aşağıdakı şəkildə aydın görmək olar.

Gördüyünüz kimi əgər biz Loopback Processing Mode`u Replace olaraq təyin etsək, bu zaman User Configuration 1 TERMINAL OU`na tətbiq olunan User Configuration 2 ilə əvəz olunur. Bu zaman ancaq TERMINAL OU`ya tətbiq olunan GPO`lar tətbiq olunur. İndi isə Loopback Merge Mode`u aktiv edib, buna baxaq.Bunun üçündə sizə rahat olsun deyə, şəkil düzəltmişəm.

Şəkildən gördüyünüz kimi bu zaman User Configuration 2 və User Configuration 1 birləşir. Bunların işləmə prinsipi çox asandır. Amma qarışıq olduğuna görə çox zaman insanlar qarışdırır. Ümid edirəm ki, bundan sonra hər şey daha da aydın olacaq. Növbəti məqalələrdə görüşənə qədər…
