Merhaba ,

Bir sebepten dolayı Biztalk Server 2010’da daha önce yaptığınız konfigurasyonu sildiniz ve yeniden yapmak istiyorsunuz. SQL Server’da SSO dahil Biztalk’a dair bütün database’leri sildiniz Configuration Wizard’da, EDI/AS2 Runtime kısmında aşağıdaki hatayı alabilirsiniz.

Deploying Activity... ERROR: The BAM deployment failed. DTS package BAM_DM_ResendJournalActivity already exists on server sql_server_fqdn.

Çözüm için :

1-) SQL Server Integration Servisini bağlanın ve aşağıdaki objeleri silin. 



2-) Ardından Biztalk sunucunuza gidin, aşağıdaki biçimde bm.exe aracını çalıştıralım. Hem AS2 reporting definition’larını hem edi’ı silmeyi unutmayın. Yani iki komutta aynı görünüyor ama xml dosyaların ismine dikkat ediniz lütfen.   



Ardindan Configuration Wizard`ı tekrar çalıştırın ve işleminizi başarı ile tamamlayın.
