Introdução


Uma parte importante da manutenção de um serviço é garantir sua recuperação em caso de falhas e ataques. O backup é uma peça fundamental em qualquer plano de recuperação em qualquer solução e o Lync Server não é diferente.

O backup de um pool Lync Server é muito mais simples que os backups das versões anteriores. No Lync todas as informações de configurações de servidores, serviços e políticas são armazenadas no Central Management Store (CMS) e o Lync Server Management Shell possui cmdlet para realizar backups e restore de vários componentes do Lync Server. Outras informações como o Location Services e Response Group são armazenadas em bases de dados SQL Server. As informações sobre os contatos e grupos dos usuários são armazenadas em outra base (Rtc.mdf) e possui uma ferramenta específica para seu backup



Backup das Configurações da Topologia


Para realizar o backup das configurações do Central Management Store execute no shell do Lync Server:

    

Export-CsConfiguration -FileName <caminho para o arquivo backup>



Backup do Location Information Services


Para realizar o backup do Location Information Services execute no shell do Lync Server:


Export-CsLisConfiguration -FileName <caminho para o arquivo backup>




Backup do Response Group


Para realizar o backup do Response Group execute em todo Lync Server que execute os serviços do Response Group:

Instale o Lync Server 2010 Resource Kit tools e localize o script RgsImportExport.ps1 

No shel navegue até a pasta onde esta localizado o script e execute 


Import-Module .\RgsImportExport.ps1 


Export-CsRgsConfiguration <serviço:poolFQDN> -FileName <caminho para o arquivo backup>

Backup do File Store


Para descobrir a localização do Lync Server File Store abra o Topology Builder, faça o download da topologia atual. Na guia File stores contem a informação de qual servidor e compartilhamento utilizado pelo pool


Copie toda a pasta utilizando Robocopy ou outra ferramenta do sistema
