1.0. Перед началом установки

Перед началом установки PowerPivot ознакомьтесь со следующими действиями, которые необходимо выполнить.

1.1. Установка PowerPivot завершается ошибкой, если не установлена среда выполнения VSTO 4.0

Перед началом установки PowerPivot необходимо установить средства Visual Studio для Office (VSTO) 4.0. Загрузить среду выполнения VSTO можно со страницы 1.1. Усe:11pt;color:#0000ff;">центра загрузки Майкрософт . Эта проблема распространяется только на пользователей Office 2010. С выходом Office 2010 с пакетом обновлений 1 (SP1) отдельная загрузка среды выполнения VSTO не требуется.

1.2. На компьютере должна быть установлена платформа .NET Framework 4.0

Если на компьютере отсутствует платформа .NET Framework 4.0, версия SQL Server 2012 программы PowerPivot для Excel установится успешно, но в ленте Excel не будет видна вкладка PowerPivot. Необходимо установить платформу .NET Framework 4.0.

1.3. На компьютере необходимо установить Silverlight 5

PowerPivot для SharePoint и Power View включают компоненты на основе Silverlight, для которых требуется, чтобы на компьютере пользователя было установлено приложение Silverlight 5. Если требуемая версия Silverlight отсутствует, то при переходе к той части веб-приложения, которая использует Silverlight, программа предложит установить Silverlight. Silverlight 5 можно установить с сайта http://go.microsoft.com/fwlink/?LinkId=243096.

1.4. Для PowerPivot для SharePoint требуется SharePoint Server 2010 с пакетом обновления 1 (SP1)

Если планируется использовать функции SQL Server 2012 в среде SharePoint, необходимо установить SharePoint Server 2010 с пакетом обновления 1 (SP1). Также рекомендуется установить последние обновления для SharePoint Server 2010 с пакетом обновления 1 (SP1).

Решение. Необходимо установить пакет обновления 1 (SP1) и полностью применить обновления на ферме серверов перед началом добавления к ферме компонентов SQL Server. Это требование распространяется на следующие функции SQL Server 2012.

· Использование экземпляра компонента ядра СУБД в качестве сервера базы данных фермы.

· Настройка PowerPivot для SharePoint.

· Развертывание служб Reporting Services в режиме интеграции с SharePoint.

1.5. Ранее созданные рабочие книги PowerPivot для SharePoint не обновляются автоматически при установке SQL Server 2012

При обновлении существующей установки PowerPivot для SharePoint до версии SQL Server 2012 программы PowerPivot для SharePoint существующие рабочие книги PowerPivot для SharePoint по умолчанию не обновляются автоматически. Без обновления рабочие книги можно использовать в SharePoint, но обновление данных для рабочих книг SQL Server 2008 R2 PowerPivot будет завершаться ошибкой. На рабочие книги SQL Server 2012 PowerPivot обновление не влияет. 2.0. Известные проблемы

Microsoft SQL Server 2012 PowerPivot для SharePoint касаются следующие известные проблемы.

2.1. Если установлен флажок «Кроме того, как можно скорее обновить данные», запланированное обновление данных не работает

SQL Server 2012 PowerPivot для SharePoint не выполняет обновление рабочих книг, у которых в расписании обновления данных имеется флажок «Кроме того, как можно скорее обновить данные». Как только пользователь нажмет кнопку «ОК» в расписании обновления данных, PowerPivot для SharePoint обновит рабочие книги, но во всех последующих циклах обновления рабочие книги обновляться не будут. После первого обновления рабочие книги больше не обновляются. Журнал обновления данных не покажет никаких последующих попыток обновления.

2.2. Управление пузырьковой диаграммой панели управления PowerPivot может не работать

На панели управления PowerPivot управление пузырьковой диаграммой не работает, если региональные настройки сервера SharePoint выглядят следующим образом:

· Пострадавший формат (появляет_работать">· Непострадавший формат (не появляется ошибка): месяц/день/год, год/месяц/день (например, США, Япония и т. д.)

2.3. При создании BISM-файла с использованием PowerPivot для SharePoint проверка может завершиться сбоем

Файл соединения семантической модели бизнес-аналитики (BISM-файл) выполняется в разных средах безопасности при его проверке и использовании. Для использования этого файла учетная запись службы приложения должна иметь разрешения администратора сервера службы SSAS для целевого сервера. Но при проверке используется учетная запись службы веб-приложения SharePoint. Поэтому нередко встречается ситуация, в которой пользователь, создающий BISM-файл, может не быть в веб-приложении, имеющем разрешения на проверку, но, когда файл действительно используется, он находится в таком веб-приложении. Чтобы убедиться, что при сбое проверки возникла именно такая ситуация, просто установите флажок «Сохранить файл ссылки без проверки подключения» и нажмите кнопку «ОК». Убедитесь в том, что приложение, в котором должен использоваться BISM-файл, например службы Excel или Reporting Services, не имеет нужных разрешений (т. е. учетная запись службы имеет разрешения администратора сервера службы SSAS на целевом сервере), а затем попытайтесь использовать файл. В действительности BISM-файл може�� работать даже при сбое проверки.
