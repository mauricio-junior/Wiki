Introdução:

Vejo muitos usuários com esse mesmo problema, quando vai tentar fazer acesso remoto em determinado usuário retorna com o erro:

"Remote Desktop Disconnected" Devido a um erro de protocolo, esta sessão será desconectada. Por favor, tente conectar ao computador remoto novamente.

Irei tentar ajudar, siga abaixo o processo de resolução.

Aplica-se


Windows Server 2008 R2  e Windows Server 2008

Sintomas.

Quando um usuário remoto controla sessão de outro usuário usando Remote Desktop Services Manager ou Gerenciador de Serviços de Terminal baseado no Windows Server 2008, e depois encerra o controle remoto, uma ou ambas as sessões estão desconectados com o seguinte erro:

"Remote Desktop Disconnected" Devido a um erro de protocolo, esta sessão será desconectada. Por favor, tente conectar ao computador remoto novamente.

Causa.

Esse problema ocorre se o Remote Desktop Connection Client versão 6.0.6001 ou 6.0.6002 é utilizado com a configuração de mais alta compressão do RDP. O Windows Server 2008 SP1/SP2 por padrão usa uma configuração mais baixa compressão RDP e, portanto, só apresentam os sintomas o client com a versão acima, se a configuração foi alterada para "otimizado para usar menos largura de banda de rede", que é o máximo. Windows Server 2008 R2 usa por padrões a configuração máxima de compressão RDP.

Resolução.

Altere a configuração de compressão RDP no servidor para "Saldos de memória e largura de banda de rede" (recomendado) ou "otimizado para usar menos memória" usando a Diretiva de Grupo. Se estiver usando o Windows Server 2008 R2 você pode também escolher "Não use um algoritmo de compressão RDP".

Uma solução alternativa é a utilização de uma versão diferente da área de trabalho remota cliente de conexão do que aqueles mencionados acima, no entanto, isso pode não ser prático. Abaixo estão as instruções para fazer a mudança para a política de grupo local; se preferir você pode usar uma política de grupo de domínio.

Windows Server 2008 R2

1. Logon para o computador Desktop Services Remote Host Session como um administrator

2. Iniciar - Executar gpedit.msc

3. No painel esquerdo, em Configuração do computador, navegue para o seguinte:

Modelos Administrativos \ Componentes do Windows \ Remote Desktop Services \ Host da Sessão da Área de Trabalho Remota \ Ambiente sessão remota

4. No painel direito, dê um duplo clique no algoritmo de compressão Set para RDP data

5. Selecione Ativado e escolha Saldos de memória e rede bandwidth

6. Clique em OK para salvar a alteração

Modelos Administrativos \ Componentes do Windows \ Remote Desktop Services \ Host da Sessão da Área de Trabalt:19px;"> Windows Server 2008 (SP1 ou SP2)

1. Logon para o computador do Terminal Services como uma administrator

2. Iniciar - Executar gpedit.msc, clique em Continuar se for solicitado por UAC

3. No painel esquerdo, em Configuração do computador, navegue para o seguinte:

Modelos Administrativos \ Componentes do Windows \ Serviços de Terminal \ Terminal Server \ Ambiente sessão remota

4. No painel direito, dê um duplo clique no algoritmo de compressão Set para RDP data

5. Selecione Ativado e escolha Saldos de memória e rede bandwidth

6. Clique em OK para salvar a alteração

Passos para reproduzir

Windows Server 2008 R2 x64

1. Instale 2008 R2 Standard (Instalação Completa) x64 de DVD usando defaults

2. Após o primeiro logon, role a janela inicial de configuração de tarefas e clique em Ativar Desktop remoto.

3. Escolha Permitir conexões de computadores que estejam executando qualquer versão da Área de Trabalho Remota (menos seguro), clique OK

4. Abra o Gerenciamento do computador e criar um usuário de teste, torná-los um membro do grupo Usuários Remoto Desktop, na guia Controle Remoto, desmarque a opção "Exigir permissão do usuário"

5. Ligue a partir de um PC usando a versão do cliente Remote Desktop 6.0.6001 ou 6.0.6002 para o servidor de logon, como administrator

6. Ligue a partir de um PC usando a versão do cliente Remote Desktop 6.0.6001 ou 6.0.6002 para o servidor, logon como o user teste.

7. Na sessão do administrador, abra o Remote Desktop Services Manager, botão direito do mouse sobre o usuário teste e escolha Controle Remoto, clique em OK

8. Depois que você está vendo o desktop do usuário de teste, pressione Ctrl-Tab para acabar com o controle remoto

Windows Server 2008 x64

1. Instale 2008 Standard (Instalação Completa) x64 de DVD usando defaults

2. Após o primeiro logon, role a janela inicial de configuração de tarefas e clique em Ativar Desktop

3 remoto. Escolha Permitir conexões de computadores que estejam executando qualquer versão da Área de Trabalho Remota (menos seguro), clique OK

4. Abra o Gerenciamento do computador e criar um usuário de teste, torná-los um membro do grupo Usuários Remoto Desktop, na guia Controle Remoto, desmarque a opção "Exigir permissão do usuário"

5. Alterar RDP algoritmo de compressão para "otimizado para usar menos largura de banda de rede" usando instruções above

6. Ligue a partir de um PC usando a versão do cliente Remote Desktop 6.0.6001 ou 6.0.6002 para o servidor de logon, como administrator

7. Ligue a partir de um PC usando a versão do cliente Remote Desktop 6.0.6001 ou 6.0.6002 para o servidor de logon, como o teste user

8. Na sessão do administrador, abra o Terminal Services Manager, botão direito do mouse sobre o usuário teste e escolha Controle Remoto, clique em OK

9. Depois que você está vendo o desktop do usuário de teste, pressione Ctrl-Tab para acabar com o controle remoto

NOTA: Este problema pode ocorrer ao se conectar a um Windows Vista ou host Windows 7 em condições semelhantes. 

Conclusão


Nesse tutorial, expliquei uma forma de corrigir o problema de incompatibilidade de cliente do Remote Desktop, para acesso remoto com o Windows Server 2008 R2 e 2008.
