1.0 Antes de Instalar

Antes de instalar o PowerPivot, leia as ações necessárias que se seguem.

1.1 A Instalação do PowerPivot Falha se o Runtime VSTO 4.0 Não Estiver Instalado

Antes de instalar o PowerPivot, tem de instalar o runtime Visual Studio Tools para o Office (VSTO) 4.0. Pode transferir o runtime VSTO a partir desta página do Centro de Transferências da Microsoft . Este é um problema apenas para os utilizadores do Office 2010. Quando o Office 2010 SP1 estiver disponível, não será necessária uma transferência em separado do runtime VSTO.

1.2 É Necessário o .NET Framework 4.0 no Computador

Se não tiver o .NET Framework 4.0 no computador, a versão do SQL Server 2012 do PowerPivot para Excel é instalada com êxito, mas o separador PowerPivot não é apresentado no friso do Excel. Tem de instalar o .NET Framework 4.0.

1.2 É Necessário o .NET Framework 4.0 no Computador

Se não tiver o .NET Framework 4.0 no computador, a versão do SQL Server 2012 do PowerPivot para Excel é instalada com êxito, mas o separador PowerPivot não é apresentado no friso do Excel. Tem de instalar o .NET Framework 4.0.

1.3 É Necessário o Silverlight 5 no Computador

Tanto o PowerPivot para SharePoint como a Vista Avançada incluem componentes baseados em Silverlight que exigem a instalação do Silverlight 5 no computador do utilizador. Se não tiver a versão do Silverlight necessária, ser-lhe-á pedido que a instale quando navegar para uma área da aplicação Web em que é obrigatória. Pode instalar o Silverlight 5 a partir de http://go.microsoft.com/fwlink/?LinkId=243096.

1.4 O PowerPivot para SharePoint Exige o SharePoint Server 2010 SP1

Se pretender utilizar as funcionalidades do SQL Server 2012 num ambiente do SharePoint, será necessário o SharePoint Server 2010 Service Pack 1 (SP1). É igualmente recomendada a instalação dos patches mais recentes do SharePoint Server 2010 SP1.

Solução: Tem de instalar o SP1 e efetuar a aplicação completa de patches no farm de servidores antes de adicionar as funcionalidades do SQL Server ao farm. Este requisito aplica-se às seguintes funcionalidades do SQL Server 2012:

· Utilizar uma instância do Motor de Base de Dados como servidor de bases de dados do farm

· Configurar o PowerPivot para SharePoint

· Implementar o Reporting Services no modo SharePoint.

1.5 Os Livros Anteriores do PowerPivot para SharePoint Não Efetuam a Atualização Automática Para o SQL Server 2012

Quando atualiza a instalação atual do PowerPivot para SharePoint para a versão do SQL Server 2012 do PowerPivot para SharePoint, os livros existentes do PowerPivot para SharePoint não são atualizados automaticamente por predefinição. Sem uma atualização, os livros podem ser utilizados no SharePoint, mas a Atualização de Dados irá falhar em livros do SQL Server 2008 R2 PowerPivot. Os livros do SQL Server 2012 PowerPivot não são afetados. No entanto, pode configurar uma funcionalidade de atualização automática numa instância do PowerPivot para SharePoint que atualiza os livros do SQL Server 2008 R2 para a versão mais recente, permitindo a Atualização de Dados nesses livros. Para mais informações, consulte Atualizar o PowerPivot para Excel e Dados PowerPivot.

Solução: Se não pretender ativar a Atualização Automática de Livros PowerPivot, tem de transferir os livros do SharePoint, atualizar os livros manualmente abrindo-os no Excel e, em seguida, carregá-los de novo no SharePoint. O livro utilizado não será atualizado e terá de efetuar também a atualização manual.

2.0 Problemas Conhecidos

Os seguintes problemas conhecidos aplicam-se ao Microsoft SQL Server 2012 PowerPivot para SharePoint.

2.1 A Atualização de Dados Agendada não funciona quando 'Atualize também logo que seja possível' está selecionado

O SQL Server 2012 PowerPivot para SharePoint ignora a atualização de livros que tenham a caixa de verificação “Atualize também logo que seja possível” ativada na agenda da Atualização de Dados. O PowerPivot para SharePoint irá atualizar o livro logo que o utilizador clique no botão OK na agenda da Atualização de Dados, mas irá ignorar todos os ciclos de atualização subsequentes. Depois da primeira atualização, o livro deixa de ser atualizado. O histórico da Atualização de Dados não apresentará mais tentativas de atualização de dados.

2.2 O Controlo de Gráficos de Bolhas do Dashboard de Gestão do PowerPivot Poderá Não Funldquo;Atualize também logo que seja possível” ativada na agenda da Atualização de Dados. O PowerPivot para SharePoint irá atualizar o livro logo que o utilizador clique no botão OK na agenda da Atualização de Dados, mas irá ignorar todos os ciclos de atualização subsequentes. Depois da primeira atualização, o livro deixa de ser atualizado. O histórico da Atualização de Dados não apresentará mais tentativas de atualização de dados.

No Dashboard de Gestão do PowerPivot, o controlo de gráficos de bolhas não funciona quando as definições regionais do servidor do SharePoint têm as seguintes especificações:

· Formato afetado (ocorrência de erro): dia/mês/ano, ano/dia/mês (Exemplos: idiomas europeus, indiano, etc.)

· Formato não afetado (sem erros): mês/dia/ano, ano/mês/dia (Exemplos: E.U.A., japonês, etc.)

2.3 Poderá ocorrer uma falha na validação ao criar um ficheiro .bism utilizando o PowerPivot para SharePoint

Um ficheiro de ligação .bism (BI Semantic Model) é executado em diferentes ambientes de segurança quando é validado e quando é utilizado. Para ser utilizado, a conta de serviço da aplicação tem de ser uma conta de administrador do Servidor SSAS no servidor de destino. Durante a validação, contudo, é utilizada a conta de serviço da aplicação Web. Por conseguinte, não é raro ocorrerem situações em que o utilizador que está a criar o ficheiro .bism seja impedido de utilizar uma aplicação Web com permissões para validar, mas, quando o ficheiro é utilizado, é obtido êxito. Para verificar se é este o caso, quando a validação falhar, basta selecionar a caixa “Guardar o ficheiro de ligação sem validação da ligação” e clicar em OK. Certifique-se de que a aplicação que irá utilizar o ficheiro .bism, por exemplo, o Excel Services ou o Reporting Services, tem as permissões necessárias (ou seja, que a sua conta de serviço é a conta de administrador do Servidor SSAS no servidor de destino) e, em seguida, tente utilizar novamente o ficheiro. O ficheiro .bism executará corretamente ainda que tenha ocorrido uma falha na validação.
