Hyper-V hostun qrafik mühitdə sazlanması (az-AZ)

:

  1. Server Manager –ə daxil oluruq.

 

 

 

 

 

2.  Naviqasiya bölməsində  Hyper-V seçirik, bundan sonra Hyper-V servisi yüklənmiş serverin adını seçib, üzərindı sağ düyməni sıxmaqla açılan listdə  Hyper-V Manager seçirik.



3. Hyper-V Manager  bölməsində Actions menyusundan Hyper-V Settings seçirik.



4. Hyper-V bölməsində,  Virtual Hard Disks seçməklə virtual maşınların yaradılması zamanı fiziki yaddaş fayllarının (VHD,VHDx) harada saxlanacağı yeri göstəririk. Virtual maşın yaradılan zaman VHD və VHDx fayllarının yerləşəcəyi ünvanı yeni yaradılan virtual maşın üçün dəyişmək olar.



5. Hyper-V Settings bölməsində  Virtual Machines seçməklə virtual maşına aid konfigurasiya fayllarının yerləçəcəyi ünvanı göstəririk.  Konfiqurasiya fayllarında virtual maşınlara ayrılmış resurslar (neçə virtual LAN əlavə olunub, CPU, RAM) və s informasiyalar yadda saxlanılır



6. Hyper-V Settings bölməsində, NUMA Spanning seçməklə NUMA imkanının aktivləşrirə bilərik.

İlk növbədə “NUMA  texnologiyası nədir?”  sualına cavab verək.

Adı halda fiziki serverin birneçə CPU-ları fiziki serverdə olan bütün  RAM-lara müraiət edə bilir və istifadə edir. NUMA texnologiyasında isə fiziki CPU ya ayrılmış Ram - lar təhkim oluna bilər ki, bu zaman həər hansısa CPU yalnız ona ayrılmış RAM-a müraciət edə biləcək. Öz növbəsində gigər CPU-lar da Numa node da olan RAM a müraciət edə bilməyəcək.  Bu isə serverin performansının artmasına gətirib çıxacaq.  NUMA fiziki və ya Hyper-V  səviyyəsində sazlana bilər. Host-da olan Virtual maşın fiziki NUMA-dan və ya virtual NUMA –dan istifadə edə bilər. Nndə, NUMA Spanning seçməklə NUMA imkanının aktivləşrirə bilərik.

İlk növbədə “NUMA  texnologiyası nədir?”  sualına cavab verək.

Adı halda fiziki serverin birneçə CPU-ları fizUMA texnologiyasına  növbəti məqalələrimizdə ətraflı nəzər yetirəcəyik.



7. Hyper-V Settings bölməsində, Live Migrations seçməklə carı hostun canlı miqrasiya funksionallığında istifadə edib-etməyəcəyini, eyni vaxtda neçə virtual maşının canlı miqrasiya oluna biləcəyiç canlı miqrasiyanin hostda olan hansi şəbəkə kartı üzərindən aparılmalı olduğu kimi sazlamaları apara bilərik.  Live Migration virtual maşınların bir host (Host- Hyper-v servisi sazlanmış əməliyyat sistemidir )dan başqa bir hosta işlək vəziyyətdə köçürülməsinə deyilir. Əgər biz Host üzərində olan VM-ların başqa hosta köçürülməsinə icazə vermək istəyiriksə və ya başqa hostdan  cari hosta VM-ların köçürülməsinə icazə vermək istəyiriksə  o zaman “Allow this server to send and receive live migrations of virtual machines” seçimini seçməliyik.Live Migration funksionallığından istifadə etmək üçün host mütləq Active Directory-yə qoşulu olmalıdır. Live migration funksionallığına  gələcək məqalələrimizdə daha ətraflı nəzər yetirəcəyik



8. Hyper-V Settings bölməsində, Storage Migrations seçməklə carı hostun saxlanc miqrasiya funksionallığında istifadə edib-etməyəcəyini, eyni vaxtda neçə virtual maşının saxlanc miqrasiya oluna biləcəyi kimi sazlamaları apara bilərik. Storage Migration  əməliyyatı virtual maşının keçırılmədən bir saxlancdan başqa saxlanca köçürülməsinə deyilir.

9. Hyper-V Settings bölməsində, Replication Configuration seçməklə Host üzərindəki virtual maşınların başqa hosta nüsxələrinin köçürülməsinə və əksinə başqa host üzərindıki virtual maşınların nüsxələrinin bu host üzərinə nüsxələrinin alınmasına icazə veririk.



Virtual Switch Manager vasitəsilə biz virtual switch –lər yarada bilərik. Hər bir şəbəkə kartı ayrılıqda bir virtual süich ola bilər

  1. Hyper-V Manager bölməsində Actions menyuundan Virtual Switch Manager seçirik.



2.  New virtual network switch seçməklə yeni virtual süitch yarada bilərik. Bu bölmədə 3 tip virtual switch növü vardır

  • Public – bu virtual switch vasitəsilə host üzərindı olan virtual maşinlar yalnız bir-birləri ilə laqə yarada bilərlər, yəni vasitəsilə virtual maşınlar fiziki şəbəkəyə qoşula bilərlər.
  • İnternal- bu virtual switch vasitəsilə host üzərindı olan virtual maşinlar yalnız bir-birləri ilə laqə yarada bilərlər, yəni fiziki şəbəkəyə müraciət imkanları olmayacaq.
  • Local- virtual switch-ə qoşulmuş VLAN kartlar yalnız Virtual maşın daxilində fəaliyyət göstərə biləcək.
