I- GIỚI THIỆU

 

Microsoft Office Web Apps 2013 là một sản phẩm cho phép xem và chỉnh sửa các tài liệu chuẩn Office của Microsoft mà không cần phải cần đặt phiên bản Microsoft Office trên các máy tính Client. Ở các phiên bản trước, sản phẩm này được cài đặt lên SharePoint Server, nhưng kể từ phiên bản 2013, đây là một sản phẩm độc lập và có thể tích hợp với các sản phẩm khác của Microsoft như SkyDrive, SharePoint, Lync 2013 và Exchange 2013.

(Hình minh họa từ technet.microsoft.com)

 

Trong bài viết này tôi sẽ hướng dẫn thao tác cài đặt Microsoft Office Web Apps 2013 và tích hợp sản phẩm này vào SharePoint Server 2013 cho phép người dùng xem và chỉnh sửa tài liệu trong khi sử dụng SharePoint 2013 mà không cần cài đặt Microsoft Office trên các máy tính Client.

 

II- TRIỂN KHAI CHI TIẾT

 

Bài LAB sử dụng 4 server:

- DC2012: Domain Controller (domain mcthub.local) chạy Windows Serrer 2012

- SQLSERVER: Domain member chạy Windows Server 2012 đóng vai trò SQL Server

- SPSERVER: Domain member chạy Windows Server 2012 đóng vai trò SharePoint Server

(Bạn có thể tích hợp 3 máy này trên một server vật lý)

- OWASERVER: Domain member chạy Windows Server 2012 đóng vai trò Office Web Apps Server

 

 

Bài LAB gồm các bước chính sau đây

1- Cài đặt và cấu hình Office Web Apps Server

2- Tạo SharePoint Web Application và Site Collection

3- Tích hợp Office Web Apps Server với SharePoint Server

 

1- Cài đặt và cấu hình Office Web Apps Server

Trên máy OWASERVER, cài đặt các thành phần cần thiết cho Office Web Apps Server bằng cách mở Windows PowerShell và nhập lệnh sau:

Trên máy OWASERVER, cài đặt các thành phần cần thiết cho Office Web Apps Server bằng cách mở Windows PowerShell và nhập lệnh sau:

Add-WindowsFeature Web-Server,Web-Mgmt-Tools,Webth,Web-App-Dev,Web-Net-Ext45,Web-Asp-Net45,Web-ISAPI-Ext,Web-ISAPI-Filter,Web-Includes,InkandHandwritingServices

 

Chờ đợi quá trình cài đặt

 

Sau khi cài đặt thành công, restart máy bằng lệnh shutdown -r -t 0

 

Cài đặt Office Web Apps Server bằng cách chạy file setup.exe từ bộ cài đặt

 

Chấp nhận thông tin bản quyền - Nhấn Continue

 

Chấp nhận thư mục cài đặt mặc định - Nhấn Install

 

Chờ đợi quá trình cài đặt

 

Sau khi cài đặt thành công, nhấn nút Close

 

Tiếp theo bạn cần cài đặt bản cập nhật cho Office Web Apps Server bằng cách download và cài đặt bản cập nhật từ Link sau

http://www.microsoft.com/en-us/download/details.aspx?id=38378

 

Sau khi cài đặt bản cập nhật, bạn cần tạo Office Web Apps Server Farm bằng lệnh sau

New-OfficeWebAppsFarm –InternalURL "http://OWASERVER" –AllowHttp -EditingEnabled

 

Tiếp tục nhập lệnh sau:
dism /online /enable-feature /featurename:IIS-ASPNET45

2- Tạo SharePoint Web Application và Site Collection



Để kiểm tra hoạt động của Office Web Apps với SharePoint, tôi sẽ tạo một Web Appliacation trên SharePoint server.



 



Trên máy DC2012, mở DNS Control tạo một HOST (A) tên là SharePoint trỏ về IP của máy SharePoint server là 172.16.1.21







 



Sang máy SPSERVER, mở SharePoint 2013 Central Administration, chọn Applcation Mamagement - Manage Web application







 



Nhấn nút New







Khung Name: Nhập tên tùy ý cho Web Application



Khung Host Header: Nhập SharePoint.mcthub.local



Nhấn OK







 



Chờ đợi quá trình tạo Web Application







 



Khi hoàn tất nhấn nút OK







 



Kiểm tra Web Application vừa tạo







 




 



Kiểm tra Web Application vừa tạo






Tiếp tho tôi cần tạo Site Collection cho Web Application. Chọn Application Management - Create site collection







 



Khung Web Application: Chọn SharePoint.mcthub.local



Khung Title: Gõ tiêu đề tùy ý







 



Khung User Name: Nhập MCTHUB\Administrator



Nhấn OK







 



Chờ đợi quá trình tạo Site Collection







 



Sau khi hoàn tất, nhấn OK







 



Đến đây bạn đã có thể truy cập vào Web Application từ các máy tính trong mạng, tuy nhiên để truy cập Web Application từ SharePoint server bạn cần chỉnh registry. Mở Registry Editor (REGEDIT)







 



Bung theo đường dẫn ở hình dưới. Bấm phải chuột lên MSV1_0, chọn New - Multi-Srting Value







 



Nhập tên cho Key là BackConnectionHostNames. Bấm phải chuột lên Key vừa tạo, chọn Modify







 



Nhập giá trị là SharePoint.mcthub.local - Nhấn OK







 



Nhấn OK xác nhận ở hộp thoại cảnh báo







 



Restart Internet Information Service (IIS) bằng lệnh iisreset /noforce







 



Kiểm tra truy cập vào Web Application bằng cách mở Internet Explorer và nhập URL SharePoint.mcthub.local







 



Kiểm tra Internet Explorer hiển thị trang SharePoint Test Site







 



3- Tích hợp Office Web Apps Server với SharePoint Server



Trên máy SPSERVER, mở SharePoint 2013 Management Shell







 



Nhập lệnh New-SPWOPIBinding -ServerName OWASERVER -AllowHTTP







 



Nhập lệnh Set-SPWOPIZone -Zone "internal-http"







 



Lần lượt nhập các lệnh sau



$config = Get-SPSecurityTokenServiceConfig



$config.AllowOAuthOverHttp = $true



$config.Update()







 



Kiểm tra hoạt động của Office Web Apps Server, mở Web Application - Chọn Document - new document - Excel workbook







 



Đặt tên tùy ý cho tài liệu - Nhấn OK







 



Kiểm tra SharePoint sẽ mở Excel Web App
