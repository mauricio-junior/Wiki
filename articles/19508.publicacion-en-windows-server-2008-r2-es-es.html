Hay un número de maneras en que un administrador puede publicar Programas RemoteApp y escritorios virtuales en Windows Server 2008 R2. El siguiente contiene una descripción breve, alto nivel de las editorial opciones que tienes, dependiendo del tamaño de su implementación.

Despliegue de muy pequeño, solo servidores Host de sesión de escritorio remoto (RD Session Host):

Si sus requerimientos de implementación son:

  • Publicar sólo Programas RemoteApp desde un único servidor de RD Session Host o solo servidores Host de sesión de RD
  • No publicar los escritorios virtuales personales o pools de escritorio virtuales

En este escenario, puede publicar Programas RemoteApp y escritorios remotos de la misma manera que los publicaron en Windows Server 2008.

En la ficha configuración de acceso Web de escritorio remoto (RD Web Access) :

· Seleccione la opción "fuentes de RemoteApp uno o más".

· En el cuadro nombre de la fuente , escriba el nombre del servidor Host de sesión de RD (o si usted tiene una granja de servidores, escriba el nombre de granja RD Session Host).

Implementación de medio, varios servidores:

Si sus requerimientos de implementación son:

  • Publicar sólo Programas RemoteApp desde varios servidores Host de sesión de RD.
  • No publicar los escritorios virtuales personales o pools de escritorio virtuales.

Esta es una nueva característica en Windows Server 2008 R2, donde puede especificar múltiples servidores Host de sesión de RD delimitados por punto y coma en el campo de nombre de fuente.

En la ficha Configuración de RD Web Access :

  • Seleccione la opción "fuentes de RemoteApp uno o más".
  • En el cuadro nombre de la fuente , escriba los nombres de servidor de RD Session Host delimitados por punto y coma.

Gran despliegue, múltiples servidores y VDI:

Si sus requerimientos de implementación son:

  • Publicar Programas RemoteApp desde varios servidores Host de sesión de RD
  • Publicar personales escritorios virtuales para usuarios específicos
  • Publicar piscinas escritorio virtuales

Esta es una nueva característica en Windows Server 2008 R2, en donde puedes apuntar el servidor de RD Web Access a un remoto escritorio agente de conexión (que tiene el funcionamiento de servicio centralizado editorial como parte de ella), que a su vez agregados los Programas RemoteApp publicados desde varios servidores Host de sesión de RD, el escritorio virtual personal asignado para la solicitud del usuario de ADy las granjas de máquina virtual publicados.

  • En el cuadro nombre de la fuente , escriba el nombre del servidor RD agente de conexión.
