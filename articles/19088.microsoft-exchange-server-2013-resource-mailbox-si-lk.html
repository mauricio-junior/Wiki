Microsoft Exchange Server
<>තුල අඩංගු සාමාන්‍ය Mailbox වලට අමතරව තවත් විශේෂිත වූ Mailbox වර්ගයක් අඞංගුවන අතර, ඒවා “Resource Mailbox” නැතහොත් “උපකරණ email ගබඩාව “ යන අර්ථකථනය දිය හැකියි. එදිනෙදා රැස්වීම් කටයුතුවලදී අප නිතර භාවිතා කරනු ලබන රැස්වීම් ශාලා, Projectors, White boards සහ එවැනි අනෙකුත් උපාංග මෙලෙස “Resource Mailbox” නැතහොත් “උපකරණ email ගබඩාව “ ලෙස ක්‍රියාත්මක කිරීමේ හැකියාව ඇත.

එමගින් රැස්වීමකට අවශ්‍ය සියළුම දෑ එක්රැස් කරගැනීම මෙන්ම අනෙකුත් රැස්වීම් සමග ගැටීම් හෝ උපකරණ ලබාගැනීමේ අසීරුතාවලට මුහුණදීම් අවම කරගත හැකියි.

මාගේ අභිප්‍රාය වනුවේ Resource Mailbox  සඳහා Microsoft Exchange Server  2013 තුල සිදුකොට ඇති නවීකරණ සහ එම අංගය ක්‍රියාත්මක වන ආකාරය පහදා දීමයි.

මූලික කරුණු

Microsoft Exchange Server  2013 තුල Resource Mailbox  සැදීම සඳහා ඔබගේ Microsoft  Active Directory ය තුල  තමන් භාවිතා කරන  User Account එක අවම වශයෙන්   Recipient Management Group (Minimum Permission)  තුළවත් අන්තර්ගත වියයුතු වනවා..

ඉන් අනතුරුව ය තුල  තමන් භාවිතා කරන  User Account එක අවම වශයෙන්   Recipient Management Group (Minimum Permission) Resource Mailbox  සැදීම සඳහා Exchange Administration Console (EAC) භාවිතා කල හැකියි. තවද මෙම කටයුත්ත අපට Exchange PowerShell අනුසාරයෙන් ද සිදුකල හැකියි.

ඔබ Exchange PowerShell භාවිතා කරන්නේ නම් පහත සඳහන් නියෝගය ක්‍රියාත්මක කිරීම තුලින් සිදුකල හැකියි.

පහතින් සඳහන් වන්නේ Exchange Administration Console (EAC) භාවිතා කොට මෙය සකසන ආකාරයයි.



ඉහත රුපයේ දැක්වෙන ආකාරයට Room mailbox නියෝගය මගින් අපගේ කටයුත්ත ආරම්භ වනවා.



ඉහත රුපයේ සඳහන් ආකාරයට මුලින්ම අපට මෙම Room mailboxයට නමක් සඳහන් කලයුතු අතර එය තේරුමක් සහිත දුටු පමණින් හඳුනාගත හැකි නමක් විය යුතුය.

දෙවනුව මෙම Room mailbox එක සඳහා email address එකක් ලබාදිය යුතු වනවා. මගේ උපදේශය වන්නේ මේ සඳහා public email address යොදා නොගන්නා ලෙසයි. එයට හේතුව වන්නේ, පිටත පුද්ගලයෙකුට මෙම Room mailbox හරහා යම්කිසි හානියක් කිරීමේ අවදානමක් පවතින බැවිනි. ඵ් සඳහා පළපුරුදු කෙනෙකුගේ උපදෙස් ලබාගන්නේ නම්, Room mailbox සඳහා public email address යොදා ගැනීමේ හැකියාව ඇත.

ඉන් අනතුරුව මෙම Room mailbox ය Active Directory ය තුල   ගොනු ගත කිරීමට අවශ්‍ය නම් ඵ් සඳහා යොමු කිරීම සිදු කල හැකියි.

ඉන් අනතුරුව වැදගත් අංගය වනුයේ, මෙම රැස්වීම් ශාලාව තුල අසුන් ගැනීමේ ධාරිතාවය (Capacity) සඳහන් කිරීමයි. එය හුදෙක් මෙය භාවිතා කරන්නන් හට මෙම ශාලාවේ ධාරිතාව සඳහන් කිරීම පමණක් සිදු කෙරේ.

අනෙකුත් වැදගත් අංගය වන්නේ, මෙම රැස්වීම් ශාලාව ලබාගැනීම සඳහා ලැබෙන අයැදුම් ලබා ගන්නා ආකාරය සැකසීමයි.

  • Accept or Decline booking request automaticallyමෙය ආකාර දෙකකින් සිදු කල හැකියි.

ඔබ මෙම අංගය භාවිතා කලහොත් මෙම Room mailbox වෙත ලැබෙන රැස්වීම් ශාලාව ලබාගැනීම සඳහා ලැබෙන අයැදුම් ස්වයංක්‍රීයව එම රැස්වීම් ශාලාවේ Room Availability මත සිදුවේ.

  • Select Delegates who can Accept or Decline booking request

ඔබ මෙම අංගය භාවිතා කලහොත් මෙම Room mailbox වෙත ලැබෙන රැස්වීම් ශාලාව ලබාගැනීම සඳහා ලැබෙන අයැදුම් පිළිගැනීම හෝ ප්‍රතික්ෂේප කිරීම ඔබ ආයතනයේ සිටින කෙනෙකු ලවා සිදුකල හැකියි. මෙමගින් සිදුවන්නේ ඔබ රැස්වීම් ශාලාව ලබාගැනීම සඳහා අයැදුමක් යැවූ විට එම අයැදුම පළමුව එම රැස්වීම් ශාලාවේ භාරකරු වෙත ලබා දීමත්, ඉන්පසු එම භාරකරු හට එම රැස්වීම් ශාලාව ලබාදීම හෝ ප්‍රතික්ෂේප කිරීමත් සිදුවනවා .

අනෙකුත් වැදගත් අංග වනුයේ, මෙම Room mailbox ස්ථාන ගතකරනු ලබන mailbox database සඳහන් කිරීමයි. මේ සඳහා ඔබගේ Exchange Server තුල ඇති ඕනෑම mailbox database එකක් භාවිතා කල හැකියි.

තවද Room mailbox සියලුම දෙනාගෙන් වසන් කොට අවශ්‍ය පුද්ගලයින් කිහිප දෙනෙකුට පමණක් භාවිතා කිරීමට අවශ්‍ය නම් ඒ සඳහා අපට Address Book Policy භාවිතා කල හැකියි.

අනතුරුව මෙහි අඩංගු අනෙකුත් අංග සකසුරුවම් කිරීමට එහි ඇති Properties අංගය ක්‍රියාත්මක කිරීම තුලින් කල හැකියි.

පළමුව General අංශයේ ඇති සාමාන්‍ය විස්තර ලබාදීම කළහැකි අතර ඉන් වඩාත් :16px;"> සියලුම දෙනාගෙන් වසන් කොට අවශ්‍ය පුද්ගලයින් කිහිප දෙනෙකුට පමණක් භාවිතා කිරීමට අවශ්‍ය නම් ඒ සඳහා අපට Address Book Policy

දෙවනුව delegates අංශය අප කලින් සාකච්ඡා කල නිසා , එය ක්‍රියා කරන ආකාරය මෙම ලිපියෙහි ඉහළින් සඳහන් කර ඇත.



තෙවනුව
