Il s'agit d'un script powershell qui active le "Developer Dashboard" dans une ferme SharePoint :

$consoleSettings =[Microsoft.Sharepoint.Administration.SPWebService]::ContentService.DeveloperDashboardSettings
$consoleSettings.DisplayLevel = "OnDemand"
$consoleSettings.RequiredPermissions = "EmptyMask"
$consoleSettings.TraceEnabled = $true
$consoleSettings.Update()

Court mais efficace, maintenant dès que vous parcourez votre site SharePoint, vous verrez une icône de diagnostics à côté de votre nom d'utilisateur dans la barre supérieure.

Dans le cas où vous recevez des erreurs disant que PowerShell n'a pas trouvé le type Microsoft.Sharepoint.Administration.SPWebService, chargez la dll de SharePoint à l'aide de la commande suivante :

[void][System.Reflection.Assembly]::LoadWithPartialName("Microsoft.SharePoint")
