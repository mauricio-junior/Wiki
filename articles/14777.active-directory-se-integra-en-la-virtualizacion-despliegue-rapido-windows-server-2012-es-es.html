Este despliegue rápido, consiste en utilizar aquello que hacemos todos, en cualquier entorno de virtualización, por ejemplo, imaginando que tenemos un Windows Server 2008 ya instalado, que por ejemplo tiene aplicadas las características de servidor de archivos, se genera una planilla, para después clonarlo, creando otro nuevo Server, que rápidamente pondremos en producción. Esto es un poco lo que se pretende con los Controladores de Dominio virtualizados, en Windows Server 2012.

Hasta ahora en Windows Server 2008 R2, el proceso de despliegue era bastante costos y nos llevaba su tiempo en poder implementar mas DCs en un entorno virtual, porque teníamos como máximo una imagen del sistema operativo y luego teníamos que hacer todo el proceso de promocionarlos… En Windows Server 2012, vamos a poder reducir este tiempo de despliegue sustancialmente.

Windows Server 2012, nos da la oportunidad, que consiste en clonar un Controlador de Dominio. Lo que se hace realmente, es crear una copia exacta del controlador de dominio que tenemos actualmente, previa configuración (o creación) de un fichero que va a establecer los cambios a realizar sobre controlador de dominio virtual. Es decir, vamos ha hacer que un DC, el que nosotros decidamos, (“siempre hay que establecer previamente que controlador de dominio es el que se va a poder clonar, por medidas de seguridad”), y el resultado de clonar este DC, vendrá tras aplicar el fichero que previamente hemos definido de cómo queremos esa clonación.

Esto cubre dos necesidades:

1.- La recuperación frente a desastres y mejorar los rendimientos del sistema y/o plataforma.

2.- Fácil despliegue sobre las famosas nubes privadas.

 

Requisitos:

    - Esto solo lo podemos hacer con Controladores de Dominio de Windows Server 2012, solo con hipervisores que sean compatibles con el identificador de generación (VM-Generation-ID)

    - Nuestro PDC tiene que ser un Windows Server 2012 para autorizar operaciones de clonado.

    - El DC a clonar debe de pertenecer al grupo “Clonate Domain Cntrollers”, grupo que dará autorización para ser clonado.

    - Generar el fichero: DCCloneConfig.XML para la clonación que debe de estar presente en el DC clonado.

 à     Respecto a estos requisitos previos, tenemos que tener en cuenta, los servicios que están instalados en los Controladores de Dominio. Los servicios mas comunes que comparten maquina con los DC son los servidores de DNS, servidores de archivo FRS, DFSR que no tienen ningún problema para ser clonados. Otros servicios o aplicaciones que pudiera tener el Controlador de Dominio no estará soportado su clonaje…
